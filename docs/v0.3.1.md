# leaferGame v0.3.1

## 版本定位

`v0.3.1` 是 `0.3.x` 的第一批落地：

**Collision Semantics Pass**

这一版的目标不是重写碰撞系统，而是先让当前最小碰撞能力从：

- 只有 “当前是否碰撞”

升级到：

- `enter`
- `stay`
- `exit`

---

## 版本目标

`v0.3.1` 完成后，碰撞系统至少应该具备：

- 当前碰撞查询
- 新发生碰撞查询
- 持续碰撞查询
- 结束碰撞查询
- layer 过滤仍然可用

---

## 范围边界

这一版要做：

- `enter/stay/exit` 语义
- 基于现有 AABB 的状态比较
- 查询 API 扩展
- 最小测试覆盖

这一版不做：

- spatial partition
- trigger / solid 分离
- collision callback bus
- 复杂碰撞响应

---

## 建议 API

在保留现有 API 的同时，增加：

- `hasCollisionEnter()`
- `getCollisionEnter()`
- `hasCollisionStay()`
- `getCollisionStay()`
- `hasCollisionExit()`
- `getCollisionExit()`

---

## 完成标准

- 同一对实体第一次接触时能出现在 `enter`
- 连续多帧保持接触时能出现在 `stay`
- 分离后的下一次 flush 中能出现在 `exit`
- 原有 `hasCollision()` / `getCollisions()` 语义保持兼容

---

## 验收清单

发布 `v0.3.1` 前，至少确认：

- `enter/stay/exit` 是否都有自动化测试
- layer 过滤是否对新 API 生效
- sample 或最小测试是否能证明语义符合预期
