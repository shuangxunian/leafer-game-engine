# leaferGame v0.5.4

## Version Positioning

`v0.5.4` is the fourth `0.5.x` delivery pass:

**Component Schema and Validation Metadata Pass**

This version gives common framework components a small, readable schema surface that tooling and future editors can consume.

---

## Goals

After `v0.5.4`, the project should support:

- component schema metadata
- a schema registry
- default schemas for common framework components
- stable lookup errors for missing schemas
- field metadata that can later drive inspector/editor controls

---

## Scope

This version includes:

- `ComponentSchema`
- `ComponentSchemaField`
- `ComponentSchemaRegistry`
- `createDefaultComponentSchemaRegistry()`
- default schemas for:
  - `transform`
  - `size`
  - `collider`
  - `velocity`

This version does not include:

- editor UI
- runtime mutation from schema
- full JSON schema compatibility
- custom validation execution
- schema-driven entity template parsing

---

## Semantics

- Schemas are plain metadata.
- Schema ids align with entity template component type ids.
- Field order is stable.
- Unknown schema ids fail clearly through `require(...)`.
- `get(...)` returns `undefined` for optional lookups.
- The default registry covers data-only framework components first.

---

## Done When

- Default schemas can be listed.
- Common component schemas can be looked up by id.
- Missing required schemas throw stable errors.
- Field metadata exposes type, required/default state, and descriptions.
- `npm run check` and `npm test` pass.
