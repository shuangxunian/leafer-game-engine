# leaferGame v0.3.0

## 版本定位

`v0.3.0` 是下一个明确阶段：

**Engine Capability Sprint**

如果说：

- `0.1.x` 解决稳定骨架
- `0.2.x` 解决 framework 复用能力

那么 `0.3.x` 的目标就是：

- 让引擎开始具备更像“引擎能力”的语义层

---

## 阶段目标

`v0.3.0` 完成后，希望项目在下面几块明显前进一步：

- collision 不再只是“当前有没有撞上”
- camera 不再只是数据壳子
- asset system 不再只是字符串 registry
- entity 组装方式开始走向 prefab / factory
- debug/tooling 开始从 snapshot 走向 runtime 能力

## 当前优先级

`0.3.x` 里最值得先做的是 collision。

原因：

- 现有 sample 已经在用碰撞
- 它会直接影响玩法表达能力
- `enter/stay/exit` 一旦到位，后面的 trigger、hurtbox、pickup、击中反馈都会更自然

---

## 与上一阶段的关系

关系可以这样理解：

- `0.2.x` 让 framework “更像 framework”
- `0.3.x` 让 engine “更像 engine”
