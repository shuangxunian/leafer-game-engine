# leaferGame v0.5.1

## Version Positioning

`v0.5.1` is the first `0.5.x` delivery pass:

**Asset Manifest and Loading Primitives Pass**

This version introduces a small data contract for declaring assets outside hardcoded scene setup.

---

## Goals

After `v0.5.1`, the project should support:

- declaring sprite assets through a manifest-like data shape
- registering manifest assets into `AssetRegistry`
- deterministic load result data
- clear validation errors for duplicate or invalid sprite declarations
- tests for manifest registration semantics

---

## Scope

This version includes:

- `AssetManifest`
- `AssetManifestSprite`
- `AssetLoadResult`
- `AssetLoadError`
- `AssetRegistry.loadManifest(manifest)`
- `loadAssetManifest(registry, manifest)`

This version does not include:

- asynchronous image fetching
- bundle integration
- remote asset loading
- texture atlas parsing
- sprite frame animation data
- editor asset panels

---

## Semantics

- A manifest is plain data.
- Sprite ids must be non-empty.
- Duplicate sprite ids inside the same manifest fail validation.
- If validation fails, no manifest sprites are registered.
- The load result reports registered ids and validation errors.
- Existing `registerSprite`, `requireSprite`, and legacy `register(id, source)` behavior remains valid.

---

## Done When

- A sprite manifest can register multiple sprite assets.
- Invalid sprite ids produce stable errors.
- Duplicate sprite ids produce stable errors.
- Failed manifests do not partially mutate the registry.
- Existing asset registry tests still pass.
- `npm run check` and `npm test` pass.
