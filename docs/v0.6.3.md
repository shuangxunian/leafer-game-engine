# leaferGame v0.6.3

## Version Positioning

`v0.6.3` is the third `0.6.x` delivery pass:

**Entity Inspector Panel Section Pass**

This version turns inspector output from appended plain text into a named tooling panel section that can be rendered consistently in browser tooling.

---

## Goals

After `v0.6.3`, the project should support:

- a pure data representation for tooling panel sections
- a dedicated entity inspector section
- stable formatting for entity rows and component summaries
- browser panel rendering that visually separates runtime and inspector data
- a small foundation for later collapsible or interactive inspector UI

---

## Scope

This version includes:

- `ToolingPanelSection` data shape
- section builders for runtime debug and entity inspector output
- browser rendering for panel section titles and lines
- tests for section formatting stability

This version does not include:

- selecting an entity
- expanding/collapsing rows
- editing component values
- schema-driven field controls
- persistence for panel layout

---

## Semantics

- Existing plain text formatters remain available.
- Section builders are pure functions and can be tested without a DOM.
- `BrowserToolingPanel` renders section data instead of one large text block.
- The entity inspector section focuses on scanability: scene summary, entity rows, and component rows.
- Future schema and interaction work should build on top of these sections instead of replacing them.

---

## Done When

- A tooling snapshot can be converted into panel sections.
- Inspector data appears under a dedicated `Entity Inspector` section.
- The browser panel renders each section with a heading and body.
- Existing debug and tooling formatters remain compatible.
- `npm run check`, `npm test`, and `npm run build:example` pass.
