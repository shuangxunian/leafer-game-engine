# leaferGame v0.4.4

## Version Positioning

`v0.4.4` is the final `0.4.x` delivery pass:

**Tooling Public API Cleanup Pass**

This version makes the new debug and inspector capabilities easier to consume from a stable public tooling entrypoint.

---

## Goals

After `v0.4.4`, the project should support:

- a simple aggregate tooling snapshot API
- clear public imports from the tooling subpath
- README-level usage guidance for runtime tooling
- tests for the aggregate tooling snapshot shape

---

## Scope

This version includes:

- `createToolingSnapshot(scene, options)`
- `ToolingSnapshot` and `ToolingSnapshotOptions` types
- optional inclusion of scene inspector data
- README public API documentation for `@shuangxunian/leafer-game-engine/tooling`
- final `0.4.x` documentation updates

This version does not include:

- editor UI
- custom inspector schemas
- timeline or replay tooling
- browser overlay redesign
- package publishing

---

## Semantics

- `createToolingSnapshot` is a convenience API over existing debug and inspector snapshot helpers.
- Debug snapshot data is always included.
- Inspector snapshot data is opt-in.
- Existing low-level helpers remain public and valid.
- The aggregate API should not mutate scene, entities, components, systems, render nodes, or assets.

---

## Done When

- `createToolingSnapshot(scene)` returns a debug snapshot.
- `createToolingSnapshot(scene, { inspector: true })` also returns a scene inspector snapshot.
- Existing debug and inspector helpers remain usable.
- README documents the tooling public API.
- `npm run check`, `npm test`, and `npm run build:example` pass.
