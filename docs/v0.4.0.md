# leaferGame v0.4.0

## Version Positioning

`v0.4.0` starts the next stage:

**Tooling and Editor-Friendly Sprint**

The previous stages made the engine skeleton, framework layer, and engine-facing features more usable. This stage focuses on the things that make development feel inspectable: debug overlays, runtime snapshots, scene inspection, and eventually editor-friendly structure.

---

## Stage Goals

`0.4.x` should move the project toward:

- visible runtime debug information
- scene and entity inspection
- collision visualization
- clearer tooling/public API boundaries
- structures that can later feed an editor

---

## Current Priority

The first useful step is a debug overlay.

Reason:

- `v0.3.5` already made debug snapshots richer
- the sample can immediately consume it
- later tooling can reuse the same snapshot and formatting path

---

## Relation to Previous Stages

- `0.2.x` made framework reuse clearer
- `0.3.x` made engine capabilities more concrete
- `0.4.x` makes runtime state easier to inspect while building content
