# leaferGame v0.20.1

## Version Positioning

`v0.20.1` is the first implementation pass in the `0.20.x` stage:

**Collision Pair Query Baseline**

This version adds structured collision pair queries to `CollisionSystem`, so downstream games can inspect current, entered, stayed, and exited collision pairs without rebuilding rect/layer metadata from raw entity arrays.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.20.1` does not add a physics engine, rigid bodies, collision resolution, trigger scripting, spatial partitioning, map-to-collider generation, visual collider editing, editor handles, or content authoring workflows. It only adds reusable runtime query data over the existing AABB collision system.

---

## Goals

After `v0.20.1`, the project should support:

- `CollisionPairSnapshot`
- `getCollisionPairs(...)`
- `getCollisionEnterPairs(...)`
- `getCollisionStayPairs(...)`
- `getCollisionExitPairs(...)`
- layer-filtered pair queries
- copied rect data so snapshots cannot mutate collider state
- deterministic pair ordering based on collision detection order

---

## Scope

This version includes:

- collision pair snapshot types in `/framework`
- current / enter / stay / exit pair query helpers
- tests for pair content, filtering, copying, and lifecycle semantics
- README/public API progress updates

This version does not include:

- collision response
- physics materials
- trigger event bus integration
- broadphase optimization
- collider authoring UI
- tooling panels

---

## Semantics

- Pair snapshots include `a` and `b` entries.
- Each entry includes `entity`, `entityId`, `entityName`, `layer`, and `rect`.
- Pair rects are copied from collider rects.
- Layer filters match if either side of the pair belongs to the requested layer.
- Pair queries expose the same lifecycle buckets as existing entity queries.

---

## Done When

- Collision pairs can be queried for current, enter, stay, and exit buckets.
- Existing `hasCollision(...)` / `getCollisions(...)` APIs stay compatible.
- Pair snapshots are copied safely.
- Public docs mention the collision pair query baseline.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
