# leaferGame v0.7.1

## Version Positioning

`v0.7.1` is the first `0.7.x` delivery pass:

**Selection-Aware Inspector Section Model**

This version adds a pure, testable way to represent selected entity state in the tooling panel section model.

---

## Goals

After `v0.7.1`, the project should support:

- optional selected entity id in tooling panel section creation
- stable selected entity summary text
- visual line-level distinction for the selected entity row
- missing-selection output when the selected id no longer exists
- no behavior change for existing unselected inspector output

---

## Scope

This version includes:

- `ToolingPanelSelection`
- selection options for `createEntityInspectorPanelSection(...)`
- selection options for `createToolingPanelSections(...)`
- tests for selected, missing, and default inspector output

This version does not include:

- browser click handling
- panel-owned selection state
- component detail panels
- schema-assisted selected component display
- runtime mutation

---

## Semantics

- Selection is optional.
- Without a selected entity id, inspector section lines remain compatible with `0.6.x`.
- If the selected entity exists, the section includes `Selected #id name`.
- If the selected entity is missing, the section includes `Selected #id missing`.
- The selected entity row uses `>` instead of `-` to make selection visible in plain text output.

---

## Done When

- Selection-aware panel section helpers exist.
- Existing default inspector section behavior remains stable.
- Selected and missing entity ids are tested.
- `npm run check`, `npm test`, and `npm run build:example` pass.
