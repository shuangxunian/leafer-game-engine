# leaferGame v0.5.3

## Version Positioning

`v0.5.3` is the third `0.5.x` delivery pass:

**Scene Config Bootstrap Pass**

This version connects asset manifests and entity templates into a minimal scene bootstrap flow.

---

## Goals

After `v0.5.3`, the project should support:

- declaring scene content through a plain data config
- loading assets from the scene config
- instantiating entity templates from the scene config
- optionally registering systems through a small system registry
- returning a structured bootstrap result

---

## Scope

This version includes:

- `SceneConfig`
- `SceneConfigSystem`
- `SceneBootstrapResult`
- `SceneSystemRegistry`
- `bootstrapSceneFromConfig(scene, config, options?)`

This version does not include:

- async scene loading
- render node creation from config
- route/scene transition config
- editor save/load
- entity hierarchy
- scene-level schema metadata

---

## Semantics

- A scene config is plain data.
- Asset manifests are loaded before entities are created.
- Entity templates are instantiated in config order.
- Systems are optional and use an explicit registry.
- Unknown systems fail clearly.
- If asset loading fails, entity/system bootstrap does not continue.
- The bootstrap result includes loaded assets, created entities, and added systems.

---

## Done When

- A scene config can register assets and create entities.
- A scene config can add systems through a registry.
- Asset manifest failures stop bootstrap before entity creation.
- Unknown systems throw stable errors.
- Existing asset manifest and entity template helpers remain valid.
- `npm run check` and `npm test` pass.
