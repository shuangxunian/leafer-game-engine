# leaferGame v0.32.2

## Scene Quick-Start Bundle Baseline

`v0.32.2` continues the `0.32.x` Quick-Start Game Kit stage.

`v0.32.0` reduced browser input bridge boot glue, and `v0.32.1`
reduced repeated scene-owned runtime system setup. This patch connects those
two helpers into one package-facing assembly point.

This version adds:

- `createSceneQuickStartBundle(scene, options?)`
- `SceneQuickStartBundle`
- `SceneQuickStartBundleOptions`

The helper first creates a scene runtime preset through
`createSceneRuntimePreset(...)`, then optionally creates browser input bridges
through `createSceneInputBridgeBundle(...)`.

When called without options, it installs the default runtime preset:

- input
- runtime services

When `inputBridges` options are provided, it creates the configured keyboard,
pointer button, and pointer position bridge bundle after runtime setup, so
downstream boot paths do not have to remember the ordering between `InputSystem`
and browser bridge creation.

---

## Example Consumption

The routed examples now consume `createSceneQuickStartBundle(...)` from their
browser boot paths:

- `examples/collect-stars` combines its existing input/collision/runtime preset
  with keyboard bridge setup
- `examples/dodge-blocks` combines input/collision preset reuse with keyboard
  and pointer button bridge setup
- `examples/pour-sort` combines input preset reuse with pointer button and
  pointer position bridge setup
- `examples/dialogue-choice` combines input preset reuse with keyboard bridge
  setup

Gameplay rules, input action maps, HUD layout, audio playback setup, audio cue
choices, scene config, dialogue prompt routing, pour-sort validation, scoring,
timers, tooling snapshots, and gameplay snapshots remain owned by the examples.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.32.2` does not add a project generator, CLI scaffold, visual editor,
launcher, gallery, template marketplace, WeChat SDK wrapper, account system,
ads, monetization, analytics service, mobile app shell, publishing workflow,
game-specific rules engine, water-sort solver, match-3 solver, galgame script
format, statechart authoring tool, UI builder, or content authoring workflow.

The helper only sequences existing runtime preset and input bridge helpers. It
does not create gameplay entities, define game rules, mutate content files,
generate a project, or introduce a new app surface.
