# leaferGame v0.6.4

## Version Positioning

`v0.6.4` is the fourth `0.6.x` delivery pass:

**Component Schema Display Pass**

This version surfaces component schema metadata in tooling snapshots and renders it as a dedicated browser panel section.

---

## Goals

After `v0.6.4`, the project should support:

- optional schema metadata in `ToolingSnapshot`
- stable schema snapshot data copied from `ComponentSchemaRegistry`
- readable component schema rows in tooling panel output
- field-level metadata display for type, required flag, default value, and description
- a foundation for future schema-driven component inspector controls

---

## Scope

This version includes:

- `ComponentSchemaSnapshot`
- schema snapshot creation through `createToolingSnapshot`
- plain formatting for component schemas
- `Component Schemas` panel section
- dodge-blocks integration using the default component schema registry
- tests for schema snapshot and panel section stability

This version does not include:

- editing component values through schema metadata
- validating live component instances against schemas
- mapping every custom gameplay component to a schema
- schema-driven form controls
- editor persistence inside this package

---

## Semantics

- Schema metadata is optional and only appears when a schema registry is provided.
- Snapshot creation copies schema and field arrays so tooling output does not expose mutable registry internals.
- Component schemas are rendered as reference metadata, not live component values.
- Entity inspector output remains responsible for runtime entity/component state.
- Later upper-layer tools can combine entity inspector rows with schema metadata to build controls.

---

## Done When

- `createToolingSnapshot(..., { schemas })` includes schema metadata.
- `formatComponentSchemaSnapshot(...)` returns stable readable lines.
- `BrowserToolingPanel` can render a `Component Schemas` section.
- The dodge-blocks example passes a default schema registry to tooling.
- `npm run check`, `npm test`, and `npm run build:example` pass.
