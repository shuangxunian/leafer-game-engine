# leaferGame v0.10.2

## Version Positioning

`v0.10.2` is the second `0.10.x` delivery pass:

**Root And Subpath Import Guidance**

This version turns the public API inventory from `v0.10.1` into clearer consumer-facing import guidance.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.10.2` does not redesign package exports, split runtime entrypoints, add editor UI, or change gameplay behavior. It clarifies how the current package boundary should be consumed.

---

## Goals

After `v0.10.2`, consumers should understand:

- when the root package import is appropriate
- when subpath imports are preferred
- which entrypoints are currently Node-safe
- which entrypoints are browser-facing
- why tests should avoid importing the root package when they only need engine logic

---

## Scope

This version includes:

- README import guidance cleanup
- `docs/public-api.md` recommended import patterns
- `v0.10.0` stage-plan link update
- roadmap recent-version link update

This version does not include:

- new public exports
- root entrypoint refactor
- browser runtime split
- package artifact checks
- example import migration
- npm publishing

---

## Semantics

- Browser apps can use the root import when they need `createBrowserRuntime(...)`.
- Node tests and pure engine logic should prefer subpath imports such as `/core`, `/framework`, and `/tooling`.
- Browser-facing imports are valid package APIs, but should be verified through browser/example builds rather than Node import smoke tests.
- This guidance reflects the current boundary; later `0.10.x` work may make the boundary more granular.
