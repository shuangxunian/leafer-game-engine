# leaferGame v0.3.4

## Version Positioning

`v0.3.4` is the fourth delivery pass in `0.3.x`:

**Asset System Pass**

The goal is not to build a complete loader pipeline yet. This version turns assets from loose strings into typed records that can be registered, looked up, and consumed by factories and render nodes.

---

## Goals

After `v0.3.4`, the project should support:

- typed sprite asset registration
- sprite asset lookup by id
- fallback-compatible sprite creation by string id
- example factories consuming registered assets
- a path toward future preload/cache behavior

---

## Scope

This version includes:

- `SpriteAsset` render type
- richer `AssetRegistry`
- sample-level asset registration
- factory consumption of asset registry
- tests for registry behavior

This version does not include:

- actual network/image preload
- atlas slicing
- audio assets
- cache invalidation
- loading progress UI

---

## Semantics

- `AssetRegistry.registerSprite()` stores sprite metadata.
- `AssetRegistry.getSprite()` returns a typed sprite asset.
- `RenderSprite.setAsset()` accepts either a string id or a typed sprite asset.
- Factories may consume an optional registry from `EntityFactoryContext`.

---

## Done When

- `player` and `hazard` assets are registered in the sample.
- Factories use registry assets when available.
- The Leafer adapter renders typed sprite fill/shape metadata.
- Tests cover sprite registration, lookup, and missing assets.
