# leaferGame v0.12.1

## Version Positioning

`v0.12.1` is the first `0.12.x` delivery pass:

**Framework Event Bus Baseline**

This version adds a small, deterministic event bus to the Node-safe `framework` entrypoint.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.12.1` does not add an editor message bus, visual editor UI, asset browser UI, DOM panel feature, network transport, analytics integration, or npm publishing. It only adds a reusable runtime communication primitive that downstream games and later engine services can consume.

---

## Goals

After `v0.12.1`, the project should support:

- subscribing to named runtime/gameplay events
- one-shot subscriptions
- explicit unsubscribe handles
- clearing listeners by event type or globally
- deterministic dispatch order
- stable event envelopes with event type, payload, and sequence metadata
- safe listener mutation during dispatch
- Node-side tests and package export coverage

---

## Scope

This version includes:

- a new framework `EventBus`
- subscription and one-shot listener APIs
- event envelope metadata
- tests for ordering, unsubscribe, once, clear, and mutation during dispatch
- README/public API updates
- `v0.12.0` roadmap linkage
- regression verification

This version does not include:

- scheduler/timer helpers
- scene-owned event bus lifecycle wiring
- tooling snapshot integration
- browser UI
- editor features
- networked events
- npm publishing

---

## Semantics

- Event dispatch is synchronous and deterministic.
- Listeners for the same event type run in subscription order.
- `once(...)` listeners are removed after their first call.
- Unsubscribing during dispatch prevents future dispatches and does not corrupt the current listener iteration.
- Subscribing during dispatch does not make the new listener run for the already in-progress event.
- Event envelopes are immutable snapshots for the dispatched event.
