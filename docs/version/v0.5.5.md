# leaferGame v0.5.5

## Version Positioning

`v0.5.5` is the final `0.5.x` delivery pass:

**Dodge Blocks Data-Driven Integration Pass**

This version proves that the content pipeline primitives can be used by the real example, not only by isolated tests.

---

## Goals

After `v0.5.5`, the project should support:

- using an asset manifest in `examples/dodge-blocks`
- using an entity template for the player data components
- keeping render-backed and behavior-backed component wiring explicit in code
- documenting which parts of the sample are data-driven
- preserving the existing gameplay behavior

---

## Scope

This version includes:

- a local `DODGE_BLOCKS_ASSET_MANIFEST`
- a local player `EntityTemplate`
- loading assets through `AssetRegistry.loadManifest(...)`
- instantiating player transform/size/collider through `instantiateEntityTemplate(...)`
- adding player view/controller components in code after template creation
- README updates for the example

This version does not include:

- hazard templates
- fully data-driven scene config
- render node creation from data
- player controller configuration schema
- editor integration

---

## Semantics

- Static player data moves into template data.
- Runtime-only objects remain in code.
- Hazard spawning stays factory-driven because it uses random runtime parameters.
- Asset registration uses the manifest path introduced in `v0.5.1`.
- The example should continue to behave like before.

---

## Done When

- Dodge Blocks loads sprite assets through a manifest.
- Dodge Blocks creates the player base entity from a template.
- Player movement, collisions, and rendering still work.
- Example docs explain the split between data and code.
- `npm run check`, `npm test`, and `npm run build:example` pass.
