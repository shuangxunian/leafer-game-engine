# leaferGame v0.9.1

## Version Positioning

`v0.9.1` is the first `0.9.x` delivery pass:

**Framework GameFlow Primitive**

This version adds a pure framework-level game-flow helper for common gameplay lifecycle phases.

---

## Goals

After `v0.9.1`, the project should support:

- a reusable `GameFlow` class
- common phases for boot, ready, running, paused, and ended
- deterministic transition results
- clear rejection for invalid transitions
- transition hooks compatible with existing flow-style HUD or system updates

---

## Scope

This version includes:

- `GameFlowPhase`
- `GameFlowTransition`
- `GameFlowTransitionReason`
- `GameFlow`
- tests for start, pause, resume, end, reset, and invalid transitions
- public framework export

This version does not include:

- dodge-blocks migration
- tooling snapshot integration
- UI widgets
- scene loading orchestration
- save/load flow

---

## Semantics

- `boot` represents pre-ready setup.
- `ready` represents content ready to start.
- `running` represents active gameplay.
- `paused` represents temporarily suspended gameplay.
- `ended` represents a completed or failed run.
- Repeating the current phase is a no-op.
- Invalid transitions return a rejected transition result and do not mutate state.
- Valid transitions notify hooks after the state changes.

---

## Done When

- `GameFlow` exists in `framework`.
- Transition semantics are covered by tests.
- Existing `StateMachine` behavior remains compatible.
- `npm run check`, `npm test`, and `npm run build:example` pass.
