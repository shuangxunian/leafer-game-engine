# leaferGame v0.5.2

## Version Positioning

`v0.5.2` is the second `0.5.x` delivery pass:

**Entity Template Data Shape Pass**

This version introduces the first data-driven entity composition primitive.

---

## Goals

After `v0.5.2`, the project should support:

- describing simple entities with template data
- instantiating templates into a scene
- a small component factory registry
- built-in support for common framework components
- stable validation errors for unknown or invalid component declarations

---

## Scope

This version includes:

- `EntityTemplate`
- `EntityTemplateComponent`
- `EntityTemplateRegistry`
- `createDefaultEntityTemplateRegistry()`
- `instantiateEntityTemplate(scene, template, options?)`
- built-in template support for:
  - `transform`
  - `size`
  - `collider`
  - `velocity`

This version does not include:

- `ViewComponent` creation from templates
- render node creation
- scene config loading
- nested prefab inheritance
- custom component schema metadata
- editor UI

---

## Semantics

- A template is plain data.
- A template creates exactly one entity.
- Components are attached in template order.
- Unknown component types fail clearly.
- Invalid component payloads fail clearly.
- Built-in components intentionally cover only data-only framework components for now.
- Render-backed components are deferred until scene config has a render-aware bootstrap boundary.

---

## Done When

- A template can create an entity with transform, size, collider, and velocity components.
- Component data is applied to the created components.
- Unknown component types throw stable errors.
- Invalid component data throws stable errors.
- Existing factory APIs remain valid.
- `npm run check` and `npm test` pass.
