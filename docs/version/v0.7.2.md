# leaferGame v0.7.2

## Version Positioning

`v0.7.2` is the second `0.7.x` delivery pass:

**Browser Panel Entity Row Selection**

This version connects the selection-aware inspector model to the browser tooling panel by making entity rows selectable in the DOM.

---

## Goals

After `v0.7.2`, the project should support:

- clickable entity rows in `BrowserToolingPanel`
- panel-owned selected entity state
- selected state preserved across snapshot refreshes
- keyboard activation for selectable entity rows
- pure row parsing tests for entity row selection behavior

---

## Scope

This version includes:

- entity row id parsing helper
- browser panel selection state
- click and keyboard handling for entity rows
- rendering section lines as individual DOM rows
- tests for row parsing semantics

This version does not include:

- selected entity detail panel
- schema-assisted component detail formatting
- component editing
- multi-selection
- selection persistence outside the panel instance

---

## Semantics

- Only top-level entity rows are selectable.
- Component rows and summary rows are not selectable.
- Clicking an entity row stores `selectedEntityId` in the panel.
- Future `update(...)` calls reuse the selected entity id when rendering sections.
- If the selected entity disappears, existing `v0.7.1` missing-selection output is used.

---

## Done When

- `BrowserToolingPanel` can select entity rows.
- Entity row parsing is covered by tests.
- Existing panel usage remains compatible.
- `npm run check`, `npm test`, and `npm run build:example` pass.
