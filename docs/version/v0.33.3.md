# leaferGame v0.33.3

## Package Metadata Export Alignment Baseline

`v0.33.3` continues the `0.33.x` stabilization stage on the road to `1.0.0`.

The previous stabilization patches locked the package artifact and public
entrypoint documentation. This patch aligns package main/types metadata with
the root export target and checks that built JavaScript plus declaration files
exist for every public export-map entry.

This version updates:

- `scripts/verify-package.mjs`
- `tests/package-boundary.test.mjs`
- `docs/api-stability-audit.md`
- `docs/public-api.md`

The package-boundary and package verification checks now:

- require package `main` and `types` metadata to match the root export targets
- require every public export-map JavaScript target to exist after build
- require every public export-map declaration target to exist after build
- keep this as release-candidate hardening before `1.0.0`

## Product Boundary

This remains frontend 2D game engine package work.

`v0.33.3` does not add new runtime API, framework API, browser runtime behavior,
project generators, CLI scaffolds, visual editors, launchers, galleries,
template marketplaces, WeChat SDK wrappers, account systems, ads, monetization,
analytics services, publishing workflows, asset manager UIs, content authoring
workflows, HUD layout engines, audio mixers, galgame scripting languages, or
game-specific rules engines.

Game-specific rules still stay in downstream examples.
