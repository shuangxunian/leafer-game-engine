# leaferGame v0.21.2

## Version Positioning

`v0.21.2` is the second implementation pass in the `0.21.x` stage:

**Audio Runtime Service Integration**

`v0.21.1` introduced Node-safe audio data contracts and `AudioRuntimeState`. This version connects that state to scene/runtime ownership so downstream games can install a reusable audio runtime system and access its state through package APIs.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.21.2` does not add Web Audio playback, decoding, waveform UI, timeline editing, mixer controls, asset browser UI, audio authoring, project persistence, or content production workflows. It only adds scene/runtime ownership for audio intent state.

This is runtime/framework infrastructure, not an editor, mixer, asset manager, or audio production surface.

---

## Goals

After `v0.21.2`, the project supports:

- an `AudioRuntimeSystem` that owns an `AudioRuntimeState`
- `addAudioRuntime(...)` for installing audio state into a scene
- `getAudioRuntime(...)` for retrieving scene-owned audio state
- injectable audio state for tests and advanced downstream ownership
- deterministic cleanup behavior for operation records when the system is destroyed
- Node-safe tests for scene integration, custom state injection, cleanup, and package exports

---

## Scope

This version includes:

- framework-level audio runtime system/helpers
- package exports from `/framework`
- docs clarifying scene/runtime ownership
- tests for system install/retrieval/cleanup behavior
- package-boundary tests for the new public API

This version does not include:

- browser audio playback
- Web Audio adapters
- audio file loading or decoding
- fade scheduling
- spatial audio
- tooling panel sections
- dodge-blocks consumption
- editor/mixer/authoring UI

---

## Semantics

- `AudioRuntimeSystem` is a regular core `System`.
- The system does not play sound directly; it owns and exposes deterministic intent state.
- `addAudioRuntime(scene, options)` installs the system and returns it.
- `getAudioRuntime(scene)` returns the scene-owned `AudioRuntimeState` when installed.
- Callers can inject an existing `AudioRuntimeState`.
- Cleanup is explicit and deterministic, with operation records cleared by default on destroy.
- The default priority places audio runtime ownership near other framework services without affecting gameplay systems.

---

## Done When

- Audio runtime state can be installed into and retrieved from a scene.
- Destroying the system clears pending operation records by default.
- Custom injected state is preserved and can opt out of cleanup if needed.
- Public docs preserve the runtime-package/not-audio-editor boundary.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
