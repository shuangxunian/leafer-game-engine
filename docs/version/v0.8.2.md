# leaferGame v0.8.2

## Version Positioning

`v0.8.2` is the second `0.8.x` delivery pass:

**Async Manifest Loading And Partial Result Handling**

This version adds an async manifest loading path that registers valid sprite metadata and then loads each sprite with explicit per-asset results.

---

## Goals

After `v0.8.2`, the project should support:

- async manifest loading through `AssetRegistry`
- partial success results across multiple sprites
- explicit loaded, failed, and skipped sprite ids
- validation errors reported alongside load results
- compatibility with existing synchronous manifest registration

---

## Scope

This version includes:

- `AsyncAssetManifestLoadResult`
- `AssetRegistry.loadManifestAsync(...)`
- `loadAssetManifestAsync(...)`
- validation-first behavior for malformed manifests
- tests for success, failure, skipped, and validation-error paths

This version does not include:

- browser `Image` loading
- parallel loading controls
- progress percentages
- retry policies
- asset dependency graphs

---

## Semantics

- Existing `loadManifest(...)` remains synchronous and compatible.
- `loadManifestAsync(...)` validates manifest data before registering or loading.
- Valid sprites are registered before loading begins.
- Each registered sprite is passed through `loadSprite(...)`.
- A failed sprite does not prevent later sprites from loading.
- The aggregate result is `ok: true` only when there are no validation errors and no failed loads.

---

## Done When

- Async manifest loading exists and is tested.
- Partial load results are deterministic.
- Existing manifest registration tests still pass.
- `npm run check`, `npm test`, and `npm run build:example` pass.
