# leaferGame v0.18.1

## Version Positioning

`v0.18.1` is the first implementation pass in the `0.18.x` stage:

**Tile Map Data Contract Baseline**

This version adds a Node-safe tile map contract with validation, defensive copying, tile lookup, and tile/world coordinate helpers.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.18.1` does not add visual map editors, tile painting tools, asset browsers, drag-and-drop level editing, save/load workflows, pathfinding, physics, rendering integration, or content production UI. It only adds reusable runtime data primitives that downstream games can consume.

---

## Goals

After `v0.18.1`, the project should support:

- defining a tile map with dimensions, tile size, and named layers
- validating basic map/layer structure before runtime use
- querying tile ids by layer and grid coordinate
- converting world coordinates to tile coordinates
- converting tile coordinates to world-space origins and bounds
- copying map/layer data so callers cannot mutate internal map state by accident

---

## Scope

This version includes:

- `TileMap` runtime helper
- `defineTileMap(...)` data-contract normalizer
- `createTileMap(...)` convenience factory
- `/framework` exports for the tile map primitives
- focused framework tests
- README/public API/roadmap progress updates

This version does not include:

- render adapter integration
- scene config integration
- collision generation from tiles
- tile asset resolution
- editor or authoring workflows
- tooling panels

---

## Semantics

- Tile map dimensions and tile sizes must be positive integers.
- Layer ids must be non-empty and unique.
- Each layer must provide exactly `width * height` tile entries.
- Tile coordinates are zero-based.
- Out-of-bounds tile queries return `undefined`.
- `worldToTile(...)` uses `Math.floor(...)`, so negative world coordinates map to negative tile coordinates and are considered out of bounds by `containsTile(...)`.
- `tileToWorld(...)` returns the top-left world-space origin for a tile.
- `getTileBounds(...)` returns top-left origin plus tile width and height.
