# leaferGame v0.7.3

## Version Positioning

`v0.7.3` is the third `0.7.x` delivery pass:

**Selected Entity Detail Section**

This version adds a dedicated read-only detail section for the selected entity in the tooling panel model.

---

## Goals

After `v0.7.3`, the project should support:

- a selected entity detail panel section
- stable detail formatting for entity state and component rows
- missing-selection detail output when the selected entity no longer exists
- automatic detail section rendering in `BrowserToolingPanel`
- no detail section when no entity is selected

---

## Scope

This version includes:

- `createSelectedEntityDetailPanelSection(...)`
- selected entity state summary lines
- selected component lifecycle and data lines
- integration with `createToolingPanelSections(...)`
- tests for selected, missing, and unselected detail output

This version does not include:

- component value editing
- schema-assisted field descriptions
- collapsible component detail rows
- multi-selection
- persistent selection across page reloads

---

## Semantics

- The detail section appears only when `selectedEntityId` is provided and inspector data exists.
- If the entity exists, the section is titled `Selected Entity`.
- If the entity is missing, the section still appears and reports `Entity #id missing`.
- Component rows show lifecycle flags and primitive component data.
- Schema-assisted detail display is deferred to `v0.7.4`.

---

## Done When

- Selected entity detail section helpers exist.
- `BrowserToolingPanel` renders detail sections after entity inspector output.
- Missing selected entities are represented clearly.
- `npm run check`, `npm test`, and `npm run build:example` pass.
