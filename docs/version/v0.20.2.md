# leaferGame v0.20.2

## Version Positioning

`v0.20.2` continues the `0.20.x` stage:

**Collision Pair Tooling Snapshot Baseline**

`v0.20.1` added structured collision pair queries to `CollisionSystem`. This version makes those pairs visible through the existing read-only tooling snapshot and panel section model, so downstream games can inspect collision runtime state without rebuilding debug summaries themselves.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.20.2` does not add a physics engine, collision response, trigger scripting, collision authoring UI, visual collider handles, map-generated colliders, editor panels, or content production workflows. It only adds read-only runtime observability over the existing AABB collision query data.

---

## Goals

After `v0.20.2`, the project should support:

- optional collision pair tooling snapshots from `createToolingSnapshot(...)`
- current / enter / stay / exit collision pair summaries
- copied rect data in tooling snapshots
- stable text formatting for collision pair state
- a browser tooling panel section for collision pair summaries
- Node-safe tests for missing-system and installed-system behavior

---

## Scope

This version includes:

- a `CollisionSnapshot` shape in `/tooling`
- `createCollisionSnapshot(scene)`
- `formatCollisionSnapshot(...)`
- `createCollisionsPanelSection(...)`
- `createToolingSnapshot(scene, { collisions: true })` integration
- README/public API/roadmap progress updates

This version does not include:

- collider editing
- collision resolution
- trigger callbacks
- physics materials
- pair selection UI
- runtime mutation controls
- map-to-collider generation

---

## Semantics

- Tooling collision snapshots are read-only summary data.
- Pair entries include entity id, entity name, layer, and copied rect data.
- Tooling snapshots do not expose mutable `Entity` references.
- If a scene has no `CollisionSystem`, the snapshot reports `installed: false`.
- Formatting preserves current / enter / stay / exit bucket names.

---
