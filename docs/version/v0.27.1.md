# leaferGame v0.27.1

## Version Positioning

`v0.27.1` is the first implementation pass in the `0.27.x` pointer-first puzzle interaction stage:

**Pointer Position Runtime Contract Baseline**

The engine already has semantic pointer button input through `InputActionMap` and `BrowserPointerButtonBridge`, but pointer-first games need more than "primary button is pressed". A water-sort, matching, grouping, slot-selection, or galgame-like interaction needs to know where the pointer is so game code can map a click to an entity, slot, region, dialogue choice, or UI affordance.

This version adds only the smallest runtime contract for pointer position.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.27.1` adds pointer position state and a browser bridge. It does not add object picking, drag-and-drop, gesture recognition, UI widgets, puzzle rules, water-sort logic, visual editor selection handles, scene graph mutation UI, galgame script systems, save/load workflows, WeChat SDK integration, launcher/gallery surfaces, or publishing workflows.

---

## Goals

After `v0.27.1`, the framework should provide:

- a Node-safe `InputSystem` pointer position state
- copied pointer position reads so consumers cannot mutate stored state
- a clear way to clear pointer position when a bridge is detached or cancelled
- a browser pointer position bridge that writes client pointer coordinates into `InputSystem`
- deterministic attach/detach listener ownership
- clear errors for invalid runtime pointer coordinates

## Semantics

- Pointer position is stored as `{ x, y }`.
- Coordinates must be finite numbers.
- `getPointerPosition()` returns a copy.
- `clearPointerPosition()` removes the current pointer position.
- The browser bridge listens to `pointermove`, `pointerdown`, and `pointerup` to keep the latest position fresh.
- The browser bridge clears position on `pointercancel`, `blur`, and `detach()`.
- The browser bridge owns only event listeners and pointer position state; it does not press/release pointer buttons.
