# leaferGame v0.11.3

## Version Positioning

`v0.11.3` is the third `0.11.x` delivery pass:

**Framework Sprite Animation Component And System**

This version connects the `v0.11.1` frame/clip contracts and `v0.11.2` playback helpers to the ECS-style framework layer.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.11.3` does not add a timeline editor, visual editor UI, asset browser UI, example gameplay integration, atlas packing, or tooling panel surface. It only adds reusable runtime behavior that downstream games and later examples can attach to entities.

---

## Goals

After `v0.11.3`, the project should support:

- a `SpriteAnimationComponent` that stores active clip and playback state
- a `SpriteAnimationSystem` that advances animation playback for live entities
- resolving clips and frames through `AssetRegistry`
- applying the current frame's sprite asset to an entity's `ViewComponent` when the render node supports sprite assets
- frame-level width/height metadata applied to the view node when present
- clear errors for missing clips, missing frames, missing sprite assets, and non-sprite render nodes
- public exports through the Node-safe `framework` package entrypoint

---

## Semantics

- `SpriteAnimationComponent` is pure ECS state and can be inspected without browser APIs.
- `SpriteAnimationSystem` owns playback advancement and asset/frame resolution.
- The system uses `AssetRegistry.requireAnimationClip(...)`, `requireSpriteFrame(...)`, and `requireSprite(...)` so missing animation data fails clearly.
- The system can update playback even if no `ViewComponent` is present.
- If a `ViewComponent` is present, its node must support `setAsset(...)` before sprite frames can be applied.
- Frame `width` and `height` update the view node when present; atlas `x/y` metadata is preserved for later but not rendered yet.

## Done When

- Animation component/system are exported from `framework`.
- ECS animation playback advances deterministically in tests.
- Current frame sprite assets are applied to sprite-capable view nodes.
- Pause/resume/stop and clip switching are covered by tests.
- Missing clip/frame/sprite/non-sprite view failures are covered by tests.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
