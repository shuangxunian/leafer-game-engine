# leaferGame v0.6.1

## Version Positioning

`v0.6.1` is the first `0.6.x` delivery pass:

**Tooling Snapshot Formatting Pass**

This version adds stable formatting for aggregated tooling snapshots before introducing a richer browser panel.

---

## Goals

After `v0.6.1`, the project should support:

- formatting `ToolingSnapshot`
- formatting `SceneInspectorSnapshot`
- readable entity/component summaries
- stable output for tests
- a pure formatting layer reusable by browser UI

---

## Scope

This version includes:

- `formatSceneInspectorSnapshot(snapshot)`
- `formatToolingSnapshot(snapshot)`
- tests for inspector/tooling formatter output

This version does not include:

- a new DOM panel
- clickable entity inspection
- component mutation
- schema display
- sample overlay replacement

---

## Semantics

- Formatting helpers return string arrays.
- Debug formatting remains backward-compatible.
- Tooling formatting includes debug lines first.
- If an inspector snapshot exists, tooling formatting appends an inspector section.
- Entity/component order follows snapshot order.

---

## Done When

- Inspector snapshots can be formatted.
- Tooling snapshots can be formatted with and without inspector data.
- Existing debug formatting remains valid.
- `npm run check` and `npm test` pass.
