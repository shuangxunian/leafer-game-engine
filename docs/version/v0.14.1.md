# leaferGame v0.14.1

## Version Positioning

`v0.14.1` is the first implementation pass in the `0.14.x` stage:

**System Observability Snapshot Hardening**

This version makes the existing read-only runtime debug snapshot more explicit about system order and lifecycle state.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.14.1` does not add editor UI, system toggle controls, component editing, scene graph editing, asset browser behavior, runtime mutation buttons, or npm publishing. It only improves read-only observability for engine runtime state.

---

## Goals

After `v0.14.1`, the project should support:

- system snapshots that expose stable registration order
- system snapshots that expose destroyed state
- a derived lifecycle label for each system
- formatted debug output that is easier to inspect without mutating runtime state
- tests that lock down deterministic system observability behavior

---

## Scope

This version includes:

- `DebugSystemSnapshot.order`
- `DebugSystemSnapshot.destroyed`
- `DebugSystemSnapshot.lifecycle`
- updated debug snapshot formatting
- tests for system order and lifecycle states
- roadmap/stage doc links

This version does not include:

- system enable/disable controls
- runtime mutation APIs
- browser-only editor panels
- component value editing
- scene or asset authoring
- npm publishing

---

## Semantics

- System order is the registration order assigned by `Scene.addSystem(...)`.
- System lifecycle is derived from existing runtime flags.
- Snapshot creation remains read-only and does not mutate scenes, systems, entities, components, render nodes, or assets.
- Formatting is allowed to become more descriptive as long as it remains deterministic.

---

## Done When

- Debug snapshots include system order, destroyed state, and lifecycle.
- Formatting exposes deterministic per-system summary lines.
- Runtime/tooling tests cover the new fields and lifecycle states.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
