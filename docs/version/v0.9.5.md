# leaferGame v0.9.5

## Version Positioning

`v0.9.5` closes the `0.9.x` delivery stage:

**Game Flow And Scene Lifecycle Docs Cleanup**

This version aligns public docs, stage links, and the dodge-blocks example description with the current game-flow and scene-lifecycle implementation.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.9.5` does not add editor UI, visual scene editing, asset authoring, drag-and-drop workflows, menu widgets, or app-shell routing. Runtime tooling and inspector language should be described as developer-facing observability, not as an editor product direction.

---

## Goals

After `v0.9.5`, the project documentation should clearly explain:

- `0.9.x` has delivered reusable game-flow and scene-lifecycle primitives
- dodge-blocks consumes those primitives as an example project
- `GameFlow` is a framework-level gameplay phase primitive
- `startSceneWithLifecycle(...)` is a runtime-level scene start helper
- tooling/inspector/debug panel features remain developer aids inside the engine package
- future editor work, if any, belongs in a separate upper-layer project or package

---
