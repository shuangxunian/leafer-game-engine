# leaferGame v0.18.3

## Version Positioning

`v0.18.3` is the third implementation pass in the `0.18.x` stage:

**Scene Config Level Declaration Integration**

This version connects the new tile map and level layout runtime primitives to scene config. Downstream games should be able to declare level/map data beside assets, entities, and systems, validate it deterministically, and receive runtime helper instances from bootstrap.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.18.3` does not add a level editor, map editor, scene graph editor, component editor, asset manager, save-file workflow, authoring UI, automatic entity generation, collision generation, rendering integration, or tooling panel. It only adds reusable runtime declarations and validation around existing level/map data contracts.

Scene config remains a bootstrap input contract, not an editor project model.

---

## Goals

After `v0.18.3`, scene config should support:

- optional `level.tileMap` declarations backed by `TileMapDefinition`
- optional `level.layout` declarations backed by `LevelLayoutDefinition`
- deterministic validation diagnostics for invalid level/map declarations
- safe validation that does not mutate scenes, entity lists, system lists, or asset registries
- bootstrap results that expose created `TileMap` and `LevelLayout` helpers when declared
- no automatic conversion from level/map data into entities, systems, render nodes, collisions, or editor state

---

## Scope

This version includes:

- `SceneConfigLevel` type support
- `SceneConfig.level` optional field
- `SceneBootstrapResult.level` optional runtime helper result
- validation diagnostics for invalid level containers, tile maps, and level layouts
- focused scene config tests
- README/public API/roadmap progress updates

This version does not include:

- example consumption
- tooling visibility for level/map data
- map rendering
- entity spawning from level declarations
- trigger/collision generation from regions
- editor or authoring workflows

---

## Semantics

- `level` is optional.
- `level.tileMap` is optional and must satisfy the existing tile map contract when present.
- `level.layout` is optional and must satisfy the existing level layout contract when present.
- Validation reports level/map problems as scene config diagnostics instead of throwing.
- Safe bootstrap returns validation errors before any runtime mutation when `validateBeforeBootstrap` is enabled.
- Successful bootstrap returns level helpers only for declarations that are present.
- Level declarations are inert runtime data. They do not create entities, systems, render nodes, collision components, or tooling state by themselves.

---

## Done When

- Scene config accepts optional level/map declarations.
- Invalid level/map declarations produce deterministic validation diagnostics.
- Safe bootstrap remains mutation-free on validation failure.
- Successful bootstrap returns `TileMap` and/or `LevelLayout` helpers without mutating scene content from level data alone.
- README/public API/roadmap mention the new `v0.18.3` work.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
