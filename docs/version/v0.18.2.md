# leaferGame v0.18.2

## Version Positioning

`v0.18.2` is the second implementation pass in the `0.18.x` stage:

**Level Spawn And Region Data Primitives**

This version adds Node-safe level metadata primitives for spawn points and rectangular regions, so downstream games can describe common level structure without hard-coding per-game helpers.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.18.2` does not add a level editor, visual region authoring, drag-and-drop tools, map painting, pathfinding, physics triggers, save/load workflows, or content production UI. It only adds reusable runtime data contracts and helpers.

---

## Goals

After `v0.18.2`, the project should support:

- defining named spawn points with world position and optional rotation
- defining named rectangular level regions with optional tags
- validating ids, numeric values, dimensions, duplicate ids, and tags
- querying spawn points and regions by id
- finding regions that contain a world point
- defensive copying so callers cannot mutate internal level metadata accidentally

---

## Scope

This version includes:

- `LevelLayout` runtime helper
- `defineLevelLayout(...)` data-contract normalizer
- `createLevelLayout(...)` convenience factory
- `/framework` exports for level metadata primitives
- focused framework tests
- README/public API/roadmap progress updates

This version does not include:

- scene config integration
- collision generation from regions
- trigger systems
- rendering or debug visualization
- editor or authoring workflows
- tooling panels

---

## Semantics

- Level layout ids, spawn ids, region ids, and region tags must be non-empty strings.
- Spawn and region ids must be unique within their own collection.
- Spawn `x`, `y`, and `rotation` must be finite numbers. Rotation defaults to `0`.
- Region `x`, `y`, `width`, and `height` must be finite numbers. Width and height must be positive.
- Region containment is inclusive on the top/left edge and exclusive on the bottom/right edge.
- Returned spawn points, regions, tags, and metadata objects are copied.

- Level layout primitives are exported from `/framework`.
- Spawn lookup, region lookup, containment, and copying behavior are covered by Node tests.
- Invalid level layout definitions fail clearly.
- README/public API/roadmap mention the new `v0.18.2` work.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
