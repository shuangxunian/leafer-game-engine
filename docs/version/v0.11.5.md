# leaferGame v0.11.5

## Version Positioning

`v0.11.5` is the fifth and final `0.11.x` delivery pass:

**Sprite Animation Stage Closeout**

This version closes the sprite animation and asset runtime sprint by documenting the public animation boundary and aligning package-facing docs with the current implementation.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.11.5` does not add editor UI, timeline editing, asset browser UI, atlas packing, render source-rect cropping, new gameplay scope, or npm publishing. It documents what the engine package now supports and what remains deliberately outside the current repository boundary.

---

## Goals

After `v0.11.5`, the project should support:

- a clear docs page for sprite animation asset/runtime concepts
- documented separation between asset metadata, playback helpers, ECS behavior, render application, example usage, and tooling visibility
- README progress aligned with the closed `0.11.x` sprint
- public API docs aligned with animation exports and tooling snapshot support
- package verification that includes any linked public docs

---

## Scope

This version includes:

- `docs/animation-runtime.md`
- `v0.11.0` stage closeout notes
- README current-progress and implemented-capability updates
- public API documentation updates
- package artifact verification updates for the new docs page
- regression verification

This version does not include:

- new runtime behavior
- new framework APIs
- example gameplay changes
- editor/timeline UI
- asset browser UI
- atlas packing or source-rect rendering
- npm publishing

---

## Semantics

- Animation docs describe the engine package boundary, not an editor workflow.
- `SpriteFrame` and `SpriteAnimationClip` remain reusable data contracts.
- Playback helpers remain deterministic and Node-safe.
- `SpriteAnimationComponent` and `SpriteAnimationSystem` remain framework/ECS runtime behavior.
- Tooling animation state remains read-only observability.
- Examples remain downstream consumers of package-style imports.

---

## Done When

- Animation asset/runtime boundary docs exist.
- README reflects that `0.11.x` is closed at `v0.11.5`.
- Public API docs point consumers to the animation runtime boundary.
- Package verification includes the new docs artifact.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
