# leaferGame v0.31.2

## Positioning

`v0.31.2` adds the second implementation slice for the UI / dialogue / scene
flow stage:

**Choice State Helper Baseline**

This version adds a small package-facing state helper for prompts that expose
available dialogue choices and need deterministic select / clear / resolve
behavior.

## Public API

The `/framework` entrypoint now exports:

- `DialogueChoiceStatePhase`
- `DialogueChoiceState`
- `DialogueChoiceStateSnapshot`
- `createDialogueChoiceState(prompt)`
- `selectDialogueChoice(state, choiceId)`
- `clearDialogueChoiceSelection(state)`
- `resolveDialogueChoiceSelection(state)`
- `getSelectedDialogueChoice(state)`
- `getResolvedDialogueChoice(state)`
- `isDialogueChoiceSelected(state)`
- `isDialogueChoiceResolved(state)`
- `getDialogueChoiceStateSnapshot(state)`

These helpers keep the current prompt choices package-facing, track the
selected choice id, validate that selections belong to the current prompt, copy
state snapshots, and return a resolved choice for downstream game-owned flow
logic.

## Boundary

This is a choice state helper baseline only. It does not add scene transitions,
story graph traversal, choice effects, visual novel scripting, branching story
editor, dialogue timeline editor, localization management system, save/load
framework, visual UI builder, launcher, gallery, marketplace, SDK wrapper,
monetization, analytics service, mobile app shell, or publishing workflow.

Story content, choice consequences, branching rules, scene switching, layout,
and visual presentation remain example-owned or downstream-game-owned until
repeated usage proves a smaller package-level helper belongs in the engine.

## Next

The next slice should add screen-space prompt rendering or example consumption
if the data and choice state helpers are sufficient.
