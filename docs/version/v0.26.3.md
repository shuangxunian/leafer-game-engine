# leaferGame v0.26.3

## Version Positioning

`v0.26.3` continues the `0.26.x` framework extraction stage:

**Actor Sprite View Attachment Helper Baseline**

Both playable examples repeatedly create a sprite render node, assign an asset, add the node to a render layer, and attach a `ViewComponent` to an existing entity.

This version extracts only that repeated package-generic render attachment step:

`attachActorSpriteView(...)`

It does not create entities, instantiate templates, define prefabs, manage spawn rules, or own gameplay state.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.26.3` adds a small runtime/framework helper for connecting an existing ECS entity to a sprite render node. It does not add a visual editor, prefab authoring UI, scene hierarchy editor, asset browser, launcher, gallery, marketplace, save/load project workflow, or content publishing workflow.

The helper is intentionally narrower than a prefab or spawn system. Actor identity, gameplay components, spawn positions, randomization, scoring, and level rules remain example-owned.

---

## Goals

After `v0.26.3`, the framework should provide:

- a small helper for attaching a sprite-backed `ViewComponent` to an existing entity
- default attachment to the `world` render layer
- optional render layer targeting
- optional sprite asset assignment
- a return value exposing both the render sprite node and created `ViewComponent`
- usage in both playable examples

---

## Scope

This version includes:

- `attachActorSpriteView(...)` in `src/framework/factory.ts`
- unit tests for default layer attachment, asset assignment, custom layer targeting, and missing render layers
- `examples/dodge-blocks` consuming the helper for player and hazard sprite views
- `examples/collect-stars` consuming the helper for player and star sprite views
- public API, README, roadmap, and package-boundary updates

This version does not include:

- a generic actor spawn system
- a prefab definition format
- scene graph authoring
- asset browser or asset management UI
- collision, animation, input, or gameplay component automation
- editor or launcher UI

---

## Semantics

- Callers pass an already-created `Entity`.
- The helper creates one render sprite through the injected `RenderAdapter`.
- If `asset` is provided, the helper assigns it to the sprite.
- The sprite is added to `renderScene.layers[layer ?? "world"]`.
- The helper adds `new ViewComponent(node)` to the entity and returns `{ node, view }`.
- Missing render layers fail with a clear error before mutating the entity.
