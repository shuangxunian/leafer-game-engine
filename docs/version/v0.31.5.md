# leaferGame v0.31.5

## Version Positioning

`v0.31.5` adds the fifth implementation slice for the UI / dialogue / scene
flow stage:

**Narrative Example Playable Flow**

This version turns `examples/dialogue-choice` from a single-prompt shell into a
small playable dialogue flow. It still adds no new public package API.

---

## Example Flow

The example now keeps its prompt content and route table locally in
`examples/dialogue-choice/dialogue-choice-scene.ts`.

The flow:

- starts at `opening`
- lets the player focus choices with keyboard actions
- resolves the selected choice through `resolveDialogueChoiceSelection(...)`
- reads the resolved choice `nextId`
- routes to the next example-owned prompt with `DialoguePromptView.setPrompt(...)`
- records visited prompt ids and last resolved choice data in the read-only
  gameplay snapshot
- reaches an `ending` prompt and marks the example flow complete
- allows reset through the example-owned reset action

This proves the existing package primitives can support a tiny interactive
narrative path without adding a package-level route table, story runtime, or
choice effect system.

---

## Package API Consumption

The example continues to consume:

- `defineDialoguePrompt(...)`
- `createDialogueChoiceState(prompt)`
- `selectDialogueChoice(state, choiceId)`
- `resolveDialogueChoiceSelection(state)`
- `getDialogueChoiceStateSnapshot(state)`
- `createDialoguePromptView(renderAdapter, renderScene, options)`
- `DialoguePromptView.setPrompt(prompt)`
- `InputActionMap`
- `BrowserKeyboardBridge`
- `GameFlow`
- `startSceneWithLifecycle(...)`

`getGameplaySnapshot()` now exposes the current prompt id, visited prompt ids,
focused choice id, last resolved choice id, last resolved next id, and complete
state.

---

## Boundary

This is an example-owned playable dialogue flow only.

It does not add a visual novel scripting language, branching story editor,
dialogue timeline editor, story graph traversal engine, reusable story format,
choice effect system, localization management, save/load framework, visual UI
builder, launcher, gallery, marketplace, SDK wrapper, account system, ads,
monetization, analytics, mobile app shell, or publishing workflow.

The prompt route table, choice effects, ending condition, and layout remain in
`examples/dialogue-choice`. The package continues to provide small
runtime/framework primitives.
