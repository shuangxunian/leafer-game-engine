# leaferGame v0.12.5

## Version Positioning

`v0.12.5` is the fifth and final `0.12.x` delivery pass:

**Runtime Services Boundary Closeout**

This version closes the runtime services and event pipeline sprint by documenting the public runtime-services boundary and aligning package-facing docs with the current implementation.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.12.5` does not add editor UI, visual scripting, runtime mutation controls, service dashboards, asset browser UI, network transport, analytics SDK integration, or npm publishing. It documents what the engine package now supports and keeps tooling framed as read-only runtime observability.

---

## Goals

After `v0.12.5`, the project should support:

- a clear docs page for runtime services concepts and usage
- documented separation between `EventBus`, `RuntimeScheduler`, `RuntimeServices`, scene integration, and tooling visibility
- README progress aligned with the closed `0.12.x` sprint
- public API docs aligned with runtime services exports and tooling snapshot support
- package verification that includes the runtime services docs artifact

---

## Scope

This version includes:

- `docs/runtime-services.md`
- `v0.12.0` stage closeout notes
- README current-progress and implemented-capability updates
- public API documentation updates
- package artifact verification updates for the new docs page
- regression verification

This version does not include:

- new runtime behavior
- new framework APIs
- example gameplay changes
- editor/service dashboard UI
- runtime mutation controls
- network transport
- analytics SDK integration
- npm publishing

---

## Semantics

- Runtime services docs describe engine package APIs, not an app framework or editor infrastructure.
- `EventBus` remains deterministic, synchronous, Node-safe, and cleanup-friendly.
- `RuntimeScheduler` remains update-driven and independent from browser timers.
- `RuntimeServicesSystem` remains opt-in per scene.
- Tooling runtime service state remains read-only observability.
- Examples remain downstream consumers of package-style imports.

---

## Done When

- Runtime services boundary docs exist.
- README reflects that `0.12.x` is closed at `v0.12.5`.
- Public API docs point consumers to the runtime services boundary.
- Package verification includes the new docs artifact.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
