# leaferGame v0.27.4

## Version Positioning

`v0.27.4` is the fourth implementation pass in the `0.27.x` pointer-first puzzle interaction stage:

**Pointer Puzzle Example Shell And Routing Baseline**

`v0.27.1` added pointer position state, `v0.27.2` added entity picking, and `v0.27.3` added source-target selection state. This version adds a small downstream-style browser example shell that consumes those pieces together.

This is example pressure testing, not a new public framework abstraction.

## Scope

This version includes:

- `examples/pour-sort` as a routed third example shell
- `?example=pour-sort` route wiring from the existing examples entry
- local pointer coordinate mapping relative to the mounted game root
- browser pointer position and pointer button bridges
- bottle entities created through existing actor/template/view helpers
- entity picking through existing rectangle/collider helpers
- source-target selection through existing framework state helpers
- HUD text showing source or source-target pair state
- package-boundary tests covering route visibility and package-style imports

This version does not include:

- water-sort rules
- moving liquid segments
- legal move validation
- win/loss progression
- undo or hints
- level authoring
- visual editor selection handles
- launcher, gallery, marketplace, SDK, account, ads, or publishing workflows

## Done When

- the example can be selected through `?example=pour-sort`
- the example uses package-style imports
- pointer position, picking, and source-target selection are wired together in the shell
- docs describe the shell as a pressure test, not a framework rule system
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass
