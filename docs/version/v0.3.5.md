# leaferGame v0.3.5

## Version Positioning

`v0.3.5` is the fifth delivery pass in `0.3.x`:

**Debug Snapshot Pass**

This version keeps tooling lightweight. It does not build an overlay UI yet; it makes the runtime snapshot richer so a future overlay has better data to show.

---

## Goals

After `v0.3.5`, debug tooling should expose:

- scene identity
- entity and system counts
- active/destroyed entity counts
- system names and priorities
- time data
- approximate FPS
- render viewport and layer names
- asset counts

---

## Scope

This version includes:

- richer `DebugSnapshot`
- optional runtime/render/asset inputs
- tests for snapshot shape
- sample boot log using runtime-aware snapshot

This version does not include:

- on-screen debug overlay
- live inspector panel
- collider visualization
- timeline or replay tooling

---

## Semantics

- `createDebugSnapshot(scene)` remains valid.
- Additional data is opt-in through an options object.
- FPS is derived from `game.time.delta` when available.
- Snapshot data should be serializable and easy to log.
