# leaferGame v0.7.0

## Version Positioning

`v0.7.0` starts the next stage:

**Interactive Runtime Inspector Sprint**

The previous stage made runtime state readable in a browser tooling panel. This stage starts turning that read-only panel into a small interactive inspector foundation.

---

## Why This Stage

`0.6.x` completed the runtime tooling panel baseline:

- formatted debug/tooling snapshots
- browser-visible panel sections
- entity inspector section
- component schema display
- dodge-blocks integration and docs cleanup

The next bottleneck is interaction:

- entity rows are visible but cannot be selected
- selected entity state has no stable representation
- panel rendering is still text-oriented
- component schemas are displayed as reference metadata but not connected to selected entities
- future editor work needs clear selection and detail-panel boundaries

`0.7.x` should add interaction primitives without becoming a full visual editor.

---

## Stage Goals

`0.7.x` should move the project toward:

- selection-aware inspector data
- selected entity summaries
- browser panel rows that can eventually be clicked
- clearer separation between inspector model and DOM rendering
- optional selected-entity detail sections
- a foundation for schema-assisted component detail views

---

## Scope Boundaries

This stage should include:

- pure selection formatting/model helpers
- lightweight panel state
- entity row click handling
- selected entity detail summaries
- tests for selection and formatting behavior

This stage should not include:

- component value editing
- drag-and-drop hierarchy editing
- persistent editor layout
- undo/redo
- external devtools protocol integration

---

## Done When

By the end of `0.7.x`:

- inspector output can represent selected entity state
- the browser panel can select an entity row
- selected entity details are visible in the example
- schema metadata can help describe selected components
- `npm run check`, `npm test`, and `npm run build:example` pass

---

## Relation to Previous Stages

- `0.5.x` made content easier to describe with data and schemas.
- `0.6.x` made runtime/tooling data easier to read.
- `0.7.x` should make runtime tooling more interactive while keeping editor ambitions scoped.
