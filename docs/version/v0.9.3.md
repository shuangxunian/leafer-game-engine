# leaferGame v0.9.3

## Version Positioning

`v0.9.3` is the third `0.9.x` delivery pass:

**GameFlow Snapshot And Tooling Visibility**

This version makes framework `GameFlow` state visible through the existing runtime debug and browser tooling surfaces.

---

## Product Boundary

This remains engine package tooling work.

`v0.9.3` exposes read-only flow state for developers and upper-layer tools. It does not add runtime mutation controls, menu widgets, visual editor UI, or app-shell routing.

---

## Goals

After `v0.9.3`, the project should support:

- debug snapshots that include current game-flow phase
- readable `GameFlow` formatting
- a dedicated tooling panel section for flow state
- dodge-blocks passing its `GameFlow` into tooling snapshots
- docs that explain the visibility layer without expanding product scope

---

## Scope

This version includes:

- `DebugGameFlowSnapshot`
- `formatDebugGameFlowSnapshot(...)`
- `createGameFlowPanelSection(...)`
- `GameFlow` support in `createDebugSnapshot(...)` and `createToolingSnapshot(...)`
- tests for snapshot shape, formatting, panel sections, and dodge-blocks-facing wiring

This version does not include:

- flow mutation controls from tooling
- UI buttons for pause/resume/restart
- save/load state
- scene loading orchestration
- editor controls

---

## Semantics

- Flow tooling is read-only.
- Snapshot data should be copied from `GameFlow` state at snapshot time.
- Runtime debug output should stay compact.
- The dedicated `Game Flow` panel section can show the current phase and gameplay-update readiness.
- Examples remain consumers of framework and tooling APIs.

---

## Done When

- `GameFlow` state appears in debug/tooling snapshots.
- Tooling panel sections include `Game Flow` when a flow is supplied.
- Dodge-blocks supplies its gameplay flow to tooling snapshots.
- `npm run check`, `npm test`, and `npm run build:example` pass.
