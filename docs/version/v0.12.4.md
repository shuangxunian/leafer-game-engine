# leaferGame v0.12.4

## Version Positioning

`v0.12.4` is the fourth `0.12.x` delivery pass:

**Runtime Services Tooling Visibility**

This version makes the runtime services introduced in `v0.12.1` through `v0.12.3` visible through read-only tooling snapshots and panel sections.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.12.4` does not add editor controls, runtime mutation buttons, visual scripting, browser-only service panels, asset browser UI, or npm publishing. It only adds read-only observability for runtime coordination state.

---

## Goals

After `v0.12.4`, the project should support:

- a structured runtime services snapshot
- formatted runtime services output
- an optional runtime services section in aggregate tooling snapshots
- a runtime services panel section for browser tooling composition
- visibility into event listener count, emitted event count, scheduler elapsed time, scheduler task count, and latest scheduler update result
- clear missing-service output when a scene has no `RuntimeServicesSystem`
- Node-side tests and package export coverage

---

## Scope

This version includes:

- `RuntimeServicesSnapshot`
- `createRuntimeServicesSnapshot(...)`
- `formatRuntimeServicesSnapshot(...)`
- `createRuntimeServicesPanelSection(...)`
- `createToolingSnapshot(scene, { runtimeServices: true })`
- tests for installed and missing runtime services snapshots
- README/public API updates
- `v0.12.0` stage progress update
- regression verification

This version does not include:

- runtime service controls
- changing event listeners from tooling
- changing scheduled tasks from tooling
- editor service containers
- browser-only implementation as the source of truth
- scene config integration
- npm publishing

---

## Semantics

- Runtime services tooling is read-only.
- Snapshot creation does not mutate the scene, services, scheduler, event bus, or systems.
- Missing runtime services produce an explicit missing snapshot instead of throwing.
- The aggregate tooling snapshot includes runtime services only when requested.
- Browser panel output is derived from the same pure snapshot/formatter APIs.
