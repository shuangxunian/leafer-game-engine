# leaferGame v0.21.1

## Version Positioning

`v0.21.1` is the first implementation pass in the `0.21.x` stage:

**Audio Data Contract Baseline**

This version should introduce Node-safe audio data contracts and runtime state helpers so downstream games can describe audio assets, cues, channels, and play/stop intent without depending on browser audio APIs.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.21.1` should not add Web Audio playback, decoding, waveform UI, timeline editing, mixer controls, asset browser UI, audio authoring, project persistence, or content production workflows. It should only add reusable runtime contracts that browser adapters and examples can consume later.

---

## Goals

After `v0.21.1`, the project should support:

- `AudioAssetDefinition`
- `AudioCueDefinition`
- `AudioChannelDefinition`
- helpers for defining/copying audio manifests
- an audio runtime state object that can record play/stop/pause/resume intent
- channel-level volume/mute state
- deterministic operation sequence numbers
- Node-safe tests for copying, validation, and state transitions

---

## Scope

This version should include:

- framework-level audio types/helpers
- package exports from `/framework`
- tests for data contracts and runtime state behavior
- README/public API/roadmap progress updates

This version should not include:

- browser audio playback
- audio file loading or decoding
- scheduler-driven fade automation
- spatial audio
- tooling panel sections
- dodge-blocks consumption

---

## Semantics

- Audio definitions should be copied defensively.
- Cue ids and asset ids should be stable strings.
- Runtime state should record intent, not play sound directly.
- Channel mute/volume changes should be deterministic and inspectable.
- Invalid ids, duplicate ids, and invalid volume values should fail clearly.

---

## Done When

- Audio contracts can be imported from `/framework`.
- Runtime audio state can be tested without DOM or Web Audio.
- Invalid definitions fail deterministically.
- Public docs mention the audio runtime primitives stage.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
