# leaferGame v0.27.5

## Version Positioning

`v0.27.5` is the fifth implementation pass in the `0.27.x` pointer-first puzzle interaction stage:

**Pointer Puzzle Playable Loop Baseline**

`v0.27.4` created the routed `pour-sort` shell. This version makes that shell playable while keeping the exact puzzle rules inside `examples/pour-sort`.

This is example pressure testing, not a new public framework abstraction.

## Scope

This version includes:

- example-owned bottle state
- a tiny top-color pour rule
- invalid move feedback for empty sources, full targets, and mismatched top colors
- rendered liquid color segments
- move count and solved-state HUD text
- example-owned read-only gameplay snapshot fields for bottle colors, move count, puzzle phase, and selected source/target
- package-boundary tests that keep the loop routed, package-facing, and example-owned

This version does not include:

- a generic puzzle rules engine
- water-sort framework APIs
- undo, hints, scoring, or progression
- level authoring
- drag/drop state
- visual editor selection handles
- launcher, gallery, marketplace, SDK, account, ads, or publishing workflows

## Done When

- `examples/pour-sort` has a small playable source-target loop
- exact puzzle rules remain in example-owned code
- no new public package API is added
- docs describe the loop as pressure testing, not framework scope
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass
