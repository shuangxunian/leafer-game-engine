# leaferGame v0.33.2

## Public Entrypoint Documentation Audit Baseline

`v0.33.2` continues the `0.33.x` stabilization stage on the road to `1.0.0`.

The previous stabilization patches recorded the public API candidate shape and
hardened package artifact verification. This patch locks the public entrypoint
documentation against the package export map so future entrypoint changes must
also update the package-facing docs.

This version updates:

- `tests/package-boundary.test.mjs`
- `docs/api-stability-audit.md`
- `docs/public-api.md`

The package-boundary tests now:

- derive package entrypoint names from `package.json.exports`
- require every exported entrypoint to be documented in `docs/public-api.md`
- require every exported entrypoint to be documented in
  `docs/api-stability-audit.md`
- keep Node-safe and browser-facing entrypoint groups explicit while preparing
  for `1.0.0`

## Product Boundary

This remains frontend 2D game engine package work.

`v0.33.2` does not add new runtime API, framework API, browser runtime behavior,
project generators, CLI scaffolds, visual editors, launchers, galleries,
template marketplaces, WeChat SDK wrappers, account systems, ads, monetization,
analytics services, publishing workflows, asset manager UIs, content authoring
workflows, HUD layout engines, audio mixers, galgame scripting languages, or
game-specific rules engines.

Game-specific rules still stay in downstream examples.
