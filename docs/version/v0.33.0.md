# leaferGame v0.33.0

## API Stability Audit Baseline

`v0.33.0` starts the `0.33.x` stabilization stage on the road to `1.0.0`.

The previous `0.32.x` stage reduced scene boot glue through quick-start
assembly helpers and a downstream-facing quick-start guide. This patch stops
expanding feature scope and records the first 1.0 API audit baseline.

This version adds:

- `docs/api-stability-audit.md`
- npm package file coverage for `docs/quick-start-game-kit.md`
- npm package file coverage for `docs/api-stability-audit.md`

The audit documents:

- the current public package entrypoints as the 1.0 candidate shape
- which entrypoints are expected to stay Node-safe
- which entrypoints remain browser-facing and opt-in
- which package-facing docs should ship in the npm package
- the change policy for the stabilization window before `1.0.0`

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.33.0` does not add new runtime API, framework API, browser runtime behavior,
project generators, CLI scaffolds, visual editors, launchers, galleries,
template marketplaces, WeChat SDK wrappers, account systems, ads, monetization,
analytics services, publishing workflows, asset manager UIs, content authoring
workflows, HUD layout engines, audio mixers, galgame scripting languages, or
game-specific rules engines.

Game-specific rules still stay in downstream examples.
