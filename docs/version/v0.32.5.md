# leaferGame v0.32.5

## Quick-Start README Example Baseline

`v0.32.5` continues the `0.32.x` Quick-Start Game Kit stage.

The previous quick-start patches added the package-facing assembly helpers for
scene runtime setup, browser input bridges, keyed HUD text, and scene-owned
audio runtime intent. This patch adds the missing downstream-facing guide that
shows how those helpers fit together.

This version adds:

- `docs/quick-start-game-kit.md`

The guide documents the intended setup path for a small browser game:

- `startSceneWithLifecycle(...)` for prepare / start boundaries
- `createSceneQuickStartBundle(...)` for runtime systems plus browser input
  bridges
- `createHudTextBundle(...)` for repeated screen-space labels
- `createSceneAudioRuntimeBundle(...)` for optional scene-owned audio intent
- explicit ownership boundaries for gameplay rules, HUD layout, audio cue
  timing, manifests, dialogue routes, scoring, timers, and snapshots

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.32.5` does not add a project generator, CLI scaffold, visual editor,
launcher, gallery, template marketplace, WeChat SDK wrapper, account system,
ads, monetization, analytics service, publishing workflow, asset manager UI,
HUD layout engine, audio mixer, galgame scripting language, game-specific rules
engine, content authoring workflow, or new app surface.

No new public runtime API is added in this patch. It only records the intended
quick-start assembly path for downstream games.
