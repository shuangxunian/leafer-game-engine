# leaferGame v0.15.4

## Version Positioning

`v0.15.4` closes the `0.15.x` stage:

**Data-Driven Scene Contract Closeout**

This version turns the scene config validation/bootstrap work into documented package-facing guidance so downstream games can consume it consistently.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.15.4` does not add editor UI, scene graph editing, component editing, asset management UI, runtime mutation controls, content publishing, remote content pipelines, or npm publishing. It only documents reusable scene config contracts and makes the documentation part of the package artifact.

---

## Goals

After `v0.15.4`, the project should support:

- a public scene config boundary document
- clear guidance for validation, safe bootstrap, and example consumption
- explicit non-editor boundaries for data-driven scene contracts
- package artifact verification that includes the new docs
- README/public API progress updates that mark `0.15.x` as closed

---

## Scope

This version includes:

- `docs/scene-config.md`
- package file list updates for the new boundary document
- package verification updates
- package-boundary tests for publish file coverage
- README/public API/roadmap updates

This version does not include:

- new runtime APIs
- async scene config validation
- save/load systems
- editor controls
- visual content authoring
- example gameplay changes

---

## Done When

- Scene config behavior is documented as a package-facing engine contract.
- The docs explain what is data-driven and what remains runtime code.
- The docs preserve the engine-package/not-editor boundary.
- Package verification includes `docs/scene-config.md`.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
