# leaferGame v0.13.3

## Version Positioning

`v0.13.3` is the third `0.13.x` delivery pass:

**Input Action Tooling Visibility**

This version makes input action mappings and current action state visible through read-only tooling snapshots and panel sections.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.13.3` does not add editor UI, keybinding settings screens, runtime rebinding controls, persistence, gamepad adapters, touch gestures, browser control panels, or npm publishing. It only adds read-only runtime observability for input action mappings.

---

## Goals

After `v0.13.3`, the project should support:

- a structured input action snapshot
- formatted input action output
- an optional input action section in aggregate tooling snapshots
- an input actions panel section for browser tooling composition
- visibility into action ids, keyboard bindings, pressed state, and just-pressed state
- useful output when no live `InputSystem` state is provided
- Node-side tests and package export coverage

---

## Scope

This version includes:

- `InputActionSnapshot`
- `createInputActionSnapshot(...)`
- `formatInputActionSnapshot(...)`
- `createInputActionsPanelSection(...)`
- `createToolingSnapshot(scene, { inputActions, input })`
- dodge-blocks passing its input action map into tooling
- tests for snapshot shape, formatting, panel section behavior, and aggregate tooling composition
- README/public API progress updates
- `v0.13.0` stage progress update
- regression verification

This version does not include:

- mutating bindings from tooling
- keybinding settings UI
- local storage or profile persistence
- gamepad or touch support
- editor controls
- npm publishing

---

## Semantics

- Input action tooling is read-only.
- Snapshot creation does not mutate `InputActionMap`, `InputSystem`, scenes, systems, or browser state.
- Action mappings can be inspected without an `InputSystem`; pressed state appears only when live input state is provided.
- The aggregate tooling snapshot includes input actions only when requested.
- Browser panel output is derived from the same pure snapshot/formatter APIs.
