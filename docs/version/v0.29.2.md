# leaferGame v0.29.2

## Version Positioning

`v0.29.2` adds the second implementation slice for the responsive Web runtime
stage:

**Browser Resize Bridge**

`v0.29.1` gave render scenes an explicit resize contract. This version adds the
opt-in browser bridge that can observe a mount/container target and call that
contract when the browser reports a size change.

---

## What Changed

- Added `BrowserResizeBridge` as a small runtime bridge around `ResizeObserver`.
- Added `createBrowserResizeBridge(...)` for direct opt-in use and deterministic
  tests with an injected observer factory.
- Added copied viewport syncing through `sync(...)`, with safe fallback to the
  current render scene viewport when a browser target reports a transient
  zero/invalid size.
- Added `resize` options to `createBrowserRuntime(...)`; when enabled, the
  runtime attaches the bridge after mount and detaches it on `stop()`.
- Kept the bridge implementation dependent only on the Node-safe
  `adapter/render-types` helpers.

---

## Product Boundary

This is browser runtime glue for a frontend 2D game engine dependency package.

`v0.29.2` does not add a responsive page builder, layout system, visual UI
designer, mobile app shell, WeChat SDK wrapper, launcher, gallery, marketplace,
account system, ads, monetization, publishing workflow, or device compatibility
matrix.

The next slice should make pointer coordinate behavior explicit after render
viewport resize.
