# leaferGame v0.21.3

## Version Positioning

`v0.21.3` is the third implementation pass in the `0.21.x` stage:

**Audio Runtime Tooling Visibility**

`v0.21.1` added audio contracts and intent state. `v0.21.2` connected that state to scene/runtime ownership. This version exposes that scene-owned audio state through read-only runtime observability so downstream games can inspect audio manifests, channel state, and operation records while debugging.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.21.3` does not add playback controls, Web Audio playback, decoding, waveform UI, timeline editing, mixer controls, volume sliders, asset browser UI, audio authoring, project persistence, or content production workflows.

It only adds read-only runtime observability over existing audio runtime state.

---

## Goals

After `v0.21.3`, the project supports:

- `createAudioRuntimeSnapshot(scene)` in `/tooling`
- copied manifest/channel/operation data in the snapshot
- a missing-audio-system snapshot state
- stable `formatAudioRuntimeSnapshot(...)` output
- an audio panel section for `BrowserToolingPanel`
- optional aggregate `createToolingSnapshot(scene, { audio: true })` inclusion
- Node-safe tests proving the snapshot is read-only and deterministic

---

## Scope

This version includes:

- tooling-level audio snapshot types/helpers
- text formatting for audio runtime state
- tooling panel section integration
- aggregate tooling snapshot opt-in
- package-boundary tests for `/tooling` exports
- README/public API progress updates

This version does not include:

- playback buttons
- channel sliders
- runtime mutation controls
- Web Audio adapters
- browser audio playback
- audio file loading or decoding
- fade scheduling
- spatial audio
- dodge-blocks consumption
- editor/mixer/authoring UI

---

## Semantics

- Audio tooling is read-only.
- Snapshot creation does not clear operations, change channels, play cues, stop cues, pause channels, or mutate manifests.
- Missing `AudioRuntimeSystem` produces a stable missing-state snapshot instead of throwing.
- Aggregate tooling includes audio only when explicitly requested.
- Panel output is compact and deterministic.

---

## Done When

- Audio runtime state can be inspected through `/tooling`.
- Aggregate tooling can include audio state by opt-in.
- Browser panel sections can display audio summary lines without mutation controls.
- Public docs preserve the runtime-package/not-audio-editor boundary.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
