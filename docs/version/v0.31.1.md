# leaferGame v0.31.1

## Positioning

`v0.31.1` adds the first implementation slice for the UI / dialogue / scene
flow stage:

**Dialogue Text / Choice Data Contract Baseline**

This version adds small package-facing data contracts for a dialogue line, a
choice, and a prompt that contains one line plus available choices.

## Public API

The `/framework` entrypoint now exports:

- `DialogueLine`
- `DialogueChoice`
- `DialoguePrompt`
- `DialoguePromptSnapshot`
- `DialogueLineInput`
- `DialogueChoiceInput`
- `DialoguePromptInput`
- `defineDialogueLine(input)`
- `defineDialogueChoice(input)`
- `defineDialoguePrompt(input)`
- `getDialoguePromptSnapshot(prompt)`

These helpers trim and validate ids/text/labels, copy prompt snapshots, and
reject duplicate choice ids inside one prompt.

## Boundary

This is a data contract baseline only. It does not add choice selection state,
scene transitions, prompt rendering, a visual novel scripting language,
branching story editor, dialogue timeline editor, localization management
system, save/load framework, story graph mutation UI, character or background
asset authoring workflow, visual UI builder, launcher, gallery, marketplace,
SDK wrapper, monetization, analytics service, mobile app shell, or publishing
workflow.

Story content, choice effects, branching rules, layout, and scene transitions
remain example-owned or downstream-game-owned until repeated usage proves a
small package-level helper belongs in the engine.

## Next

The next slice should add a small choice state helper baseline if the data
contract shape is enough.
