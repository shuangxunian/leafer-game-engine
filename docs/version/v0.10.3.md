# leaferGame v0.10.3

## Version Positioning

`v0.10.3` is the third `0.10.x` delivery pass:

**Package Artifact Verification**

This version adds a package verification command that checks the files npm would publish before release.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.10.3` does not publish a new npm version, redesign package exports, add editor UI, or change gameplay behavior. It only makes package artifacts easier to verify before publishing.

---

## Goals

After `v0.10.3`, the project should support:

- a repeatable package artifact verification command
- `npm pack --dry-run --json` based tarball inspection
- checks that exported JS and type declaration files are included
- checks that important docs and license files are included
- checks that source, tests, examples, scripts, and build-only app output are not included
- README documentation for the package verification command

---

## Scope

This version includes:

- `scripts/verify-package.mjs`
- `npm run verify:package`
- package `files` update for package-facing public API docs
- package boundary test update
- README and public API docs package verification guidance
- `v0.10.0` stage-plan link update
- roadmap recent-version link update

This version does not include:

- npm publishing
- package version bump
- root entrypoint refactor
- browser runtime split
- example import migration
- editor or asset browser work

---

## Semantics

- The verification command should run `npm pack --dry-run --json`.
- The command should fail if required package files are missing.
- The command should fail if unwanted development-only paths appear in the tarball.
- The command should verify the current export map targets, rather than duplicating a hand-written list of all built files.

---

## Done When

- `npm run verify:package` passes.
- Package artifacts include `lib`, `README.md`, `LICENSE`, `package.json`, and package-facing public API docs.
- Package artifacts exclude `src`, `tests`, `examples`, `dist`, and `scripts`.
- README explains when to run package verification.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
