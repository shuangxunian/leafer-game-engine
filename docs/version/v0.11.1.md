# leaferGame v0.11.1

## Version Positioning

`v0.11.1` is the first `0.11.x` delivery pass:

**Sprite Frame And Animation Clip Contracts**

This version adds the data contracts that later animation playback can build on.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.11.1` does not add a timeline editor, visual editor UI, asset browser UI, atlas packing, or runtime playback system. It only defines and validates the reusable data model for sprite frames and animation clips.

---

## Goals

After `v0.11.1`, the project should support:

- `SpriteFrame` metadata
- `SpriteAnimationClip` metadata
- helper functions for defining frame and clip contracts
- manifest registration for frames and clips
- registry lookup and require APIs for frames and clips
- validation for missing frame/sprite references and invalid durations

---

## Scope

This version includes:

- framework animation data contracts
- asset manifest `frames` and `clips`
- `AssetRegistry` frame/clip storage and lookup
- tests for registration, validation, and helper behavior
- `v0.11.0` stage-plan link update

This version does not include:

- animation playback state
- animation system/component behavior
- render node frame application
- atlas packing
- editor timeline UI
- example animation integration

---

## Semantics

- Sprite frames reference a registered sprite asset by `spriteId`.
- Animation clips reference registered sprite frame ids.
- Frame and clip durations are expressed in seconds.
- Manifest validation should happen before mutating the registry.

---

## Done When

- Sprite frame and animation clip contracts are exported from `framework`.
- Asset manifests can register frames and clips.
- Invalid frame/clip manifests fail without mutating the registry.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
