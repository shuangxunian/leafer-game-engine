# leaferGame v0.31.4

## Version Positioning

`v0.31.4` adds the fourth implementation slice for the UI / dialogue / scene
flow stage:

**Narrative Example Shell And Route Baseline**

This version adds a routeable `examples/dialogue-choice` downstream-style
browser example. The example consumes the existing dialogue prompt data, choice
state helper, prompt view helper, keyboard bridge, input action map, and runtime
lifecycle APIs. It does not add new public package API.

---

## Added Example Surface

The new example includes:

- `examples/dialogue-choice/boot.ts`
- `examples/dialogue-choice/dialogue-choice-scene.ts`
- `examples/dialogue-choice/input-actions.ts`
- `examples/dialogue-choice/README.md`
- `?example=dialogue-choice` route wiring from the shared examples entry

The shell renders a simple stage, character placeholder, prompt panel, dialogue
line, and three choices. Keyboard input moves the focused choice and resolves
the selected choice into the existing choice state helper.

---

## Package API Consumption

The example proves that a downstream game can already assemble a tiny narrative
screen from existing package-facing APIs:

- `defineDialoguePrompt(...)`
- `createDialogueChoiceState(prompt)`
- `selectDialogueChoice(state, choiceId)`
- `resolveDialogueChoiceSelection(state)`
- `getDialogueChoiceStateSnapshot(state)`
- `createDialoguePromptView(renderAdapter, renderScene, options)`
- `InputActionMap`
- `BrowserKeyboardBridge`
- `GameFlow`
- `startSceneWithLifecycle(...)`

The example exposes `getGameplaySnapshot()` so tests and manual checks can read
flow phase, prompt id, choice ids, focused choice, selected choice, and resolved
choice handoff without mutating the scene.

---

## Boundary

This is a narrative example shell and routing baseline only.

It does not add a visual novel scripting language, branching story editor,
dialogue timeline editor, story graph traversal, reusable story format, choice
effect system, localization management, save/load framework, visual UI builder,
launcher, gallery, marketplace, SDK wrapper, account system, ads, monetization,
analytics, mobile app shell, or publishing workflow.

The prompt content, choice effects, scene transitions, and layout choices stay
example-owned. The package continues to provide small runtime/framework
primitives that browser games can compose.
