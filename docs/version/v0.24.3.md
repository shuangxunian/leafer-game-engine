# leaferGame v0.24.3

## Version Positioning

`v0.24.3` is the third implementation pass in the `0.24.x` playable game kit stage:

**Runtime HUD Text Helper Baseline**

This version adds a small runtime HUD helper for creating screen-space text nodes through the existing render adapter and render scene layer contract. The goal is to reduce repeated example/game boilerplate for score labels, status text, and overlay prompts without creating a UI framework, visual editor, menu builder, or authoring workflow.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.24.3` does not add a visual UI editor, layout designer, drag-and-drop HUD builder, component property panel, theme system, button/widget library, UI persistence format, or content publishing workflow.

It only adds code-first runtime helpers that downstream games can call while building scenes.

---

## Goals

After `v0.24.3`, the project should support:

- a Node-safe HUD text options contract
- a helper that creates render text through `RenderAdapter`
- automatic attachment to a screen-space render scene layer
- default `ui` layer behavior with optional `overlay` support
- x/y/fontSize/visible initialization
- dodge-blocks HUD and overlay text consumption through the helper
- tests proving created text state and target layer behavior
- docs preserving the engine-package/not-editor boundary

---

## Scope

This version includes:

- `createHudText(...)` in `framework`
- public export and package-boundary coverage
- dodge-blocks title/score/status/overlay text integration
- Node tests using fake render adapter and render scene objects
- README and public API wording for runtime HUD helper direction

This version does not include:

- flex/grid/constraint layout
- buttons or interactive widgets
- text styling systems beyond direct render node fields
- menu routing
- localization
- editor UI
- visual authoring or persistence workflows

---

## Semantics

- HUD text is screen-space by default and should attach to the `ui` layer unless another screen-space layer is provided.
- Overlay prompts can opt into the `overlay` layer.
- The helper should return the created `RenderText` so game systems can keep updating text with `setText(...)`.
- The helper should initialize only simple runtime fields and should not own later updates.
- The helper should not write to world-space layers by default or introduce editor state.

---

## Done When

- Framework exports the HUD text helper.
- Dodge-blocks uses the helper for score/status/title/overlay text.
- Node tests cover default `ui` layer, explicit `overlay` layer, initial text fields, and returned render node behavior.
- Package docs explain that this is code/runtime HUD creation, not UI authoring.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
