# leaferGame v0.13.4

## Version Positioning

`v0.13.4` is the fourth and final `0.13.x` delivery pass:

**Input Actions Stage Closeout**

This version closes the input actions and control mapping sprint by documenting the public input action boundary and aligning package-facing docs with the current implementation.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.13.4` does not add editor UI, keybinding settings screens, runtime rebinding controls, persistence, gamepad adapters, touch gestures, browser control panels, new runtime APIs, example gameplay changes, or npm publishing. It documents what the engine package now supports and keeps input tooling framed as read-only runtime observability.

---

## Goals

After `v0.13.4`, the project should support:

- a clear docs page for input action concepts and usage
- documented separation between raw input state, keyboard bindings, action maps, gameplay consumption, example integration, and tooling visibility
- README progress aligned with the closed `0.13.x` sprint
- public API docs aligned with input action exports and tooling snapshot support
- package verification that includes the input actions docs artifact

---

## Scope

This version includes:

- `docs/input-actions.md`
- `v0.13.0` stage closeout notes
- README current-progress and implemented-capability updates
- public API documentation updates
- package artifact verification updates for the new docs page
- regression verification

This version does not include:

- new runtime behavior
- new framework APIs
- new tooling APIs
- example gameplay changes
- keybinding settings UI
- runtime rebinding controls
- gamepad or touch support
- npm publishing

---

## Semantics

- Input action docs describe engine package APIs, not an editor workflow.
- `InputActionMap` remains deterministic and Node-safe.
- `BrowserKeyboardBridge` remains the browser keyboard adapter that writes normalized low-level input state into `InputSystem`.
- Dodge Blocks remains a downstream-style consumer of package-style imports.
- Tooling input action state remains read-only observability.

---

## Done When

- Input action boundary docs exist.
- README reflects that `0.13.x` is closed at `v0.13.4`.
- Public API docs point consumers to the input actions boundary.
- Package verification includes the new docs artifact.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
