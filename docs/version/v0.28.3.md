# leaferGame v0.28.3

## Version Positioning

`v0.28.3` adds the third implementation slice for the real sprite / image rendering stage:

**Example Image Asset Consumption**

`v0.28.1` made the Leafer adapter image-backed, and `v0.28.2` made the framework handoff explicit. This version uses those pieces in an example so source-backed sprite rendering is no longer only covered by framework and adapter tests.

---

## What Changed

- `examples/dodge-blocks` keeps player and hazard sprite assets declared with generated image-like data URI `source` values.
- The player sprite view now uses `attachActorSpriteView(...)` with `assets + assetId`.
- Hazard factory sprite views now use the same `assets + assetId` handoff.
- The example no longer passes registry-only sprite records directly into render nodes for player/hazard views.
- `examples/dodge-blocks/README.md` documents the source-backed sprite asset consumption boundary.

This proves a downstream-style playable example can preload/register source-backed sprite assets and hand them to render views through package-facing APIs.

---

## Product Boundary

This is example integration evidence for the engine package.

`v0.28.3` does not add public package API. It does not add an asset editor, visual asset manager, atlas packer, image cropper, bundled art library, marketplace, launcher, gallery, WeChat SDK wrapper, account system, ads, monetization, or publishing workflow.

The next `0.28.x` slice should document the sprite rendering/package boundary and decide whether this stage is ready to close.
