# leaferGame v0.4.3

## Version Positioning

`v0.4.3` is the third `0.4.x` delivery pass:

**Scene Inspector Snapshot Pass**

This version turns runtime debug data into a more structured shape that can feed future inspector panels, editor tools, and richer browser overlays.

---

## Goals

After `v0.4.3`, the project should support:

- a structured scene inspector snapshot
- entity-level debug data
- component-level debug data
- stable, shallow component value extraction
- tests for inspector snapshot semantics

---

## Scope

This version includes:

- `createSceneInspectorSnapshot(scene)`
- entity snapshots with id, name, active/destroyed state, and component count
- component snapshots with name, enabled/started/destroyed state, and inspectable primitive data
- deterministic ordering based on world entity order and component order
- documentation and tests

This version does not include:

- an interactive editor panel
- component mutation from tooling
- deep object serialization
- custom per-component inspector schemas
- selection state or hierarchy editing

---

## Semantics

- Inspector snapshots are read-only data.
- Inspector snapshots should not mutate entities, components, systems, or render nodes.
- Component data extraction is intentionally shallow.
- Primitive fields are included.
- Functions, entity references, and complex object references are skipped.
- Lifecycle fields are represented explicitly instead of duplicated inside component data.

---

## Done When

- A scene can produce a structured inspector snapshot.
- Entity snapshots include id, name, active/destroyed state, and component count.
- Component snapshots include lifecycle state and shallow primitive data.
- Complex object fields are skipped safely.
- `npm run check` and `npm test` pass.
