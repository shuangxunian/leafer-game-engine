# leaferGame v0.11.4

## Version Positioning

`v0.11.4` is the fourth `0.11.x` delivery pass:

**Example Sprite Animation Integration And Tooling Visibility**

This version proves that the `v0.11.1` through `v0.11.3` animation work can be consumed from a downstream-style example and observed through read-only runtime tooling.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.11.4` does not add a visual editor, timeline editor, asset browser, content authoring workflow, atlas packing, or render source-rect cropping. It only integrates reusable engine APIs into the existing example and exposes animation state as developer-facing runtime observability.

---

## Goals

After `v0.11.4`, the project should support:

- `examples/dodge-blocks` declaring sprite frames and animation clips in its asset manifest
- the example attaching `SpriteAnimationComponent` to the player as a downstream consumer
- the example running `SpriteAnimationSystem` from the package-facing framework entrypoint
- `createToolingSnapshot(...)` optionally including read-only sprite animation state
- a browser tooling panel section that shows entity, clip, playback status, current frame, current sprite, elapsed time, and completed loops
- tests that verify animation snapshot formatting and panel section behavior

---

## Scope

This version includes:

- dodge-blocks player animation manifest data
- dodge-blocks `SpriteAnimationComponent` and `SpriteAnimationSystem` integration
- `Sprite Animations` tooling snapshot data
- `Sprite Animations` tooling panel section
- docs and README progress updates
- tests for snapshot/panel formatting and package-facing example imports

This version does not include:

- hazard animation
- atlas source rect rendering
- render adapter cropping
- timeline controls
- editor UI
- asset browser UI
- publishing to npm

---

## Semantics

- Example integration must keep package-style imports.
- Tooling animation data is read-only and derived from existing ECS component state.
- Animation tooling should not mutate components or runtime state.
- Animation tooling is optional through `createToolingSnapshot(scene, { animations: true })`.
- If no animation components exist, the section should still report a stable zero-count summary when requested.

---

## Done When

- Dodge-blocks uses sprite animation APIs as a consumer.
- Tooling snapshots can include animation state.
- Browser tooling sections can render animation state.
- README/current progress reflects `v0.11.4`.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
