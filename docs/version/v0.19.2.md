# leaferGame v0.19.2

## Version Positioning

`v0.19.2` is the second implementation pass in the `0.19.x` stage:

**Browser Pointer Button Bridge Baseline**

The previous patch added Node-safe pointer button bindings for `InputActionMap`. This version connects browser pointer button events to the existing `InputSystem`, so downstream browser games can drive semantic actions from pointer buttons through the same action map path.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.19.2` does not add pointer position tracking, drag/drop behavior, gestures, hit testing, visual input binding UI, settings screens, editor controls, asset workflows, or content authoring. It only adds a reusable browser input bridge that writes normalized pointer button state into `InputSystem`.

---

## Goals

After `v0.19.2`, the project should support:

- a browser-facing pointer button bridge
- deterministic mapping from browser pointer button numbers to `primary`, `secondary`, and `auxiliary`
- press / release / cancel cleanup for pointer button input ids
- blur cleanup to avoid stuck pointer button state
- Node-testable bridge behavior through an injectable event target
- public `/framework` export and docs for the bridge

---

## Scope

This version includes:

- `BrowserPointerButtonBridge`
- pointer button event normalization through existing pointer helpers
- tests for attach/detach, duplicate press handling, release, cancel, unsupported buttons, and blur cleanup
- README/public API/input docs progress updates

This version does not include:

- pointer coordinates
- pointer move state
- drag state
- touch gesture recognition
- gamepad support
- runtime rebinding UI
- editor or authoring workflows

---

## Semantics

- Browser button `0` maps to `pointer:primary`.
- Browser button `1` maps to `pointer:auxiliary`.
- Browser button `2` maps to `pointer:secondary`.
- Unsupported button numbers are ignored.
- Repeated `pointerdown` for an already pressed pointer button does not retrigger `justPressed`.
- `pointerup`, `pointercancel`, and `blur` release tracked pointer button state.
- `detach()` removes listeners and releases tracked pointer button state.
