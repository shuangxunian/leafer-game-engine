# leaferGame v0.31.6

## Version Positioning

`v0.31.6` closes the UI / dialogue / scene flow stage:

**UI / Dialogue / Scene Flow Closeout**

This version adds no new public package API and no new runtime behavior. It
records that the `0.31.x` baseline is complete enough for the current 1.0 path.

---

## What The Stage Leaves In Place

The completed `0.31.x` stage leaves:

- dialogue line / choice / prompt data contracts
- copied dialogue prompt snapshots
- duplicate choice id validation
- immutable dialogue choice state helpers
- selected / resolved choice handoff helpers
- copied dialogue choice state snapshots
- screen-space dialogue prompt view creation
- prompt view visibility toggling and prompt replacement
- a routed `examples/dialogue-choice` browser example
- example-owned prompt routing and playable dialogue flow
- read-only dialogue gameplay snapshots

These pieces prove a downstream browser game can assemble a small
galgame-style prototype path from package primitives without importing a story
engine.

---

## Still Out Of Scope

The closeout keeps the same boundary:

- no visual novel scripting language
- no branching story editor
- no dialogue timeline editor
- no story graph traversal engine
- no reusable story format
- no choice effect system
- no localization management workflow
- no save/load framework
- no visual UI builder
- no launcher, gallery, marketplace, SDK wrapper, account system, ads,
  monetization, analytics, mobile app shell, or publishing workflow

Prompt content, prompt route tables, choice effects, ending conditions, and UI
layout remain downstream game code.
