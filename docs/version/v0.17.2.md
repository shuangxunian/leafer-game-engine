# leaferGame v0.17.2

## Version Positioning

`v0.17.2` is the second implementation pass in the `0.17.x` stage:

**Game Tick Error Boundary Contract**

This version clarifies and tests how `Game.tick(...)` behaves when scene phases throw, especially around fixed-step accumulator ownership.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.17.2` does not add editor UI, runtime mutation controls, browser devtools, visual error overlays, app-shell routing, save/load systems, retry policies, or global error handlers. It only makes game-loop error propagation and accumulator behavior deterministic.

---

## Goals

After `v0.17.2`, the project should support:

- explicit `Game.tick(...)` error propagation semantics
- fixed-step accumulator cleanup when a fixed update throws
- tests proving update/fixed/late error boundaries
- docs that describe downstream expectations without hiding errors

---

## Scope

This version includes:

- `Game.tick(...)` fixed-step accumulator hardening
- focused core tests for update, fixedUpdate, and lateUpdate failures
- README/public API progress updates
- `v0.17.0` stage progress update

This version does not include:

- catching and swallowing runtime errors
- custom error callbacks
- retry or rollback semantics
- scene restart behavior
- browser UI for errors

---

## Semantics

- Errors thrown by `Scene.update(...)`, `fixedUpdate(...)`, or `lateUpdate(...)` still propagate to callers.
- If `Scene.update(...)` throws, fixed and late phases do not run for that tick.
- If `Scene.fixedUpdate(...)` throws, the fixed slice that was attempted is removed from the accumulator before the error propagates.
- After a fixed-update error, the next successful tick continues from the remaining accumulator normally.
- If `Scene.lateUpdate(...)` throws, update and fixed phases have already completed for that tick.
- `Game.tick(...)` does not destroy or restart the active scene on runtime hook errors.
