# leaferGame v0.22.1

## Version Positioning

`v0.22.1` is the first implementation pass in the `0.22.x` stage:

**Audio Playback Adapter Contract Baseline**

This version should introduce a Node-safe adapter contract for consuming `AudioRuntimeState` operation records without depending on Web Audio, DOM APIs, browser media elements, or real sound assets.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.22.1` does not add Web Audio playback, browser media playback, decoding, streaming audio, waveform UI, timeline editing, mixer controls, asset browser UI, audio authoring, project persistence, or content production workflows.

It only defines reusable playback adapter contracts and deterministic operation-draining semantics.

---

## Goals

After `v0.22.1`, the project should support:

- an `AudioPlaybackAdapter` interface
- adapter operation result records
- a helper for draining pending `AudioRuntimeState` operations into an adapter
- deterministic success/failure reporting
- optional preservation or clearing of drained operation records
- Node-safe tests using a fake adapter

---

## Scope

This version should include:

- framework-level playback adapter types/helpers
- package exports from `/framework`
- tests for play/stop/pause/resume/set-volume/set-muted operation dispatch
- tests for adapter failures and deterministic results
- docs clarifying this is a contract baseline, not browser playback

This version should not include:

- Web Audio implementation
- HTMLAudioElement implementation
- audio asset loading or decoding
- browser permissions/autoplay handling
- spatial audio
- tooling panel changes
- dodge-blocks playback consumption
- editor/mixer/authoring UI

---

## Semantics

- The helper should read operation records from `AudioRuntimeState`.
- It should dispatch operations to an injected adapter in sequence order.
- It should return stable result records for each attempted operation.
- It should support clearing drained operations by default.
- Adapter failures should be captured as result data instead of hiding which operation failed.
- Tests should run in Node with no DOM or media APIs.

---

## Done When

- A fake adapter can consume deterministic audio operation records.
- Operation results expose sequence, operation type, status, and optional error text.
- Public exports include the adapter contract and drain helper.
- Public docs preserve the runtime-package/not-audio-editor boundary.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
