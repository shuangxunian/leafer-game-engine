# leaferGame v0.30.5

## Positioning

`v0.30.5` closes the drag/drop and selection hardening stage:

**Drag/Drop And Selection Stage Closeout**

This version adds no new public package API. It records the completed baseline
and keeps the next work focused on package-facing runtime/framework capability
rather than editor, launcher, platform, or puzzle-template scope.

## Closed Baseline

The `0.30.x` stage now leaves these package-facing pieces in place:

- copied source-target selection snapshots and generic readiness checks
- explicit immutable source/target replacement helpers
- generic entity drag state with start/current pointer positions
- pointer-local drag delta, completion snapshots, and cancellation snapshots
- generic source-target action envelopes
- copied action snapshots and allowed/blocked validation-result envelopes
- `examples/pour-sort` consuming selection snapshots and source-target action
  helpers while keeping pour-sort rules example-owned

## Boundary

The stage did not add drop target resolution, gesture recognition, multi-touch
gameplay systems, physics simulation, water-sort validation, match-3 validation,
merge rules, inventory rules, scoring, solvers, undo/hint systems, editor drag
handles, editor selection handles, visual layout tools, launchers, galleries,
marketplaces, SDK wrappers, account systems, ads, monetization, mobile app
shells, or publishing workflows.

Game-specific rules still belong in `examples/` or downstream games. Framework
helpers should stay small, generic, deterministic, and package-facing.

## Next

The next stage should move to the next concrete 1.0 gap after this interaction
baseline, without reopening editor, launcher, marketplace, SDK wrapper,
monetization, or publishing scope.
