# leaferGame v0.30.4

## Positioning

`v0.30.4` adds the fourth implementation slice for the drag/drop and selection
hardening stage:

**Pointer Puzzle Example Hardening**

This version hardens the `pour-sort` example as a downstream consumer of the
new selection and action helpers. It does not add new public package API.

## What Changed

- `examples/pour-sort` now creates a generic source-target action from ready
  selection state via `createSourceTargetActionFromSelection("pour", state)`.
- The example records allowed/blocked action results with
  `allowSourceTargetAction(action)` and `blockSourceTargetAction(action, reason)`.
- The gameplay snapshot now includes copied action data through
  `getSourceTargetActionSnapshot(action)` plus the last action status/reason.
- The example reads selection display data through
  `getSourceTargetSelectionSnapshot(state)` while keeping the older
  `selectionPhase`, `sourceName`, and `targetName` snapshot fields compatible.

## Boundary

`v0.30.4` keeps `pourTopColor(...)`, `isPourSortSolved(...)`, bottle capacity,
move counting, invalid-move messages, solved-state logic, and liquid rendering
inside `examples/pour-sort`. It does not add water-sort validation, match-3
validation, merge rules, inventory rules, solvers, scoring, undo/hint systems,
gesture recognition, multi-touch gameplay systems, physics simulation, editor
drag handles, visual layout tools, launcher, gallery, marketplace, SDK wrapper,
monetization, or publishing workflow to the engine package.

## Next

The next slice should close the `0.30.x` drag/drop and selection hardening stage
if the current selection, drag, action, and pointer puzzle example baseline is
enough.
