# leaferGame v0.14.4

## Version Positioning

`v0.14.4` is the fourth and final `0.14.x` delivery pass:

**Runtime Observability Stage Closeout**

This version closes the runtime observability hardening sprint by documenting the public runtime observability boundary and aligning package-facing docs with the current implementation.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.14.4` does not add editor UI, system toggle controls, component editing, scene graph editing, asset management UI, dashboard product behavior, runtime mutation buttons, new runtime APIs, gameplay changes, or npm publishing. It documents what the engine package now supports and keeps runtime tooling framed as read-only observability.

---

## Goals

After `v0.14.4`, the project should support:

- a clear docs page for runtime observability concepts and boundaries
- documented separation between debug snapshots, system observability, text formatting, browser panel section formatting, aggregate tooling snapshots, and example consumption
- README progress aligned with the closed `0.14.x` sprint
- public API docs aligned with runtime observability support
- package verification that includes the runtime observability docs artifact

---

## Scope

This version includes:

- `docs/runtime-observability.md`
- `v0.14.0` stage closeout notes
- README current-progress and implemented-capability updates
- public API documentation updates
- package artifact verification updates for the new docs page
- regression verification

This version does not include:

- new runtime behavior
- new framework APIs
- new tooling APIs
- example gameplay changes
- system control UI
- editor or authoring UI
- npm publishing

---

## Semantics

- Runtime observability docs describe engine package APIs, not an editor workflow.
- Debug snapshots remain deterministic and Node-safe.
- Runtime Debug panel section formatting remains presentation-only.
- Dodge Blocks remains a downstream-style consumer of package-style imports.
- Tooling state remains read-only observability.

---

## Done When

- Runtime observability boundary docs exist.
- README reflects that `0.14.x` is closed at `v0.14.4`.
- Public API docs point consumers to the runtime observability boundary.
- Package verification includes the new docs artifact.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
