# leaferGame v0.4.2

## Version Positioning

`v0.4.2` is the second `0.4.x` delivery pass:

**Collider Visualization Pass**

This version adds the first visual debugging tool on top of the render layer structure introduced earlier.

---

## Goals

After `v0.4.2`, the project should support:

- optional collider box visualization
- collider debug nodes that live in the world layer
- automatic cleanup when entities disappear
- tests for visual debug synchronization

---

## Scope

This version includes:

- `ColliderDebugSystem`
- debug sprite creation for collider rects
- cleanup for removed entities
- documentation and tests

This version does not include:

- interactive collider editing
- collision pair labels
- trigger/solid styling
- editor selection tools

---

## Semantics

- Collider debug nodes are attached to `renderScene.layers.world`.
- Debug nodes use collider rect data, including offsets and explicit dimensions.
- Camera transforms apply naturally because debug nodes live in the world layer.
- The system is opt-in.

---

## Done When

- A collider entity produces a visible debug node.
- Moving or resizing a collider updates the debug node.
- Removing an entity destroys its debug node.
- `npm run check` and `npm test` pass.
