# leaferGame v0.26.1

## Version Positioning

`v0.26.1` is the first implementation pass in the `0.26.x` framework extraction stage:

**Bounded Position Clamp Helper Baseline**

Both `examples/dodge-blocks` and `examples/collect-stars` move a player inside a bounded playfield. They both combine semantic movement input, `limitMovementVector(...)`, entity size, bounds, and local clamp math.

This version extracts only the repeated package-generic math:

`clampPositionToBounds(...)`

It does not extract a player controller, input mapping, gameplay loop, spawn system, level system, or editor-facing construct.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.26.1` adds a small runtime math helper to the Node-safe `framework` entrypoint. It does not add a visual editor, scene builder, prefab editor, input rebinding UI, example launcher, gallery, marketplace, asset manager, save/load workflow, leaderboard, account system, or publishing workflow.

---

## Goals

After `v0.26.1`, the framework should provide:

- a pure helper for clamping a desired `{ x, y }` position to rectangular bounds
- support for optional bounds origin
- support for entity width/height
- support for symmetric padding
- deterministic behavior when the entity is larger than the available bounded area
- clear errors for invalid numeric inputs
- usage in both playable examples

---

## Semantics

- Bounds default to origin `{ x: 0, y: 0 }` when omitted.
- Padding defaults to `0`.
- Size defaults to `{ width: 0, height: 0 }`.
- Returned `x` is clamped between `bounds.x + padding` and `bounds.x + bounds.width - size.width - padding`.
- Returned `y` is clamped between `bounds.y + padding` and `bounds.y + bounds.height - size.height - padding`.
- If the available range is smaller than the entity, the helper anchors to the minimum bound.
