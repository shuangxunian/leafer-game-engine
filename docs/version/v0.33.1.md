# leaferGame v0.33.1

## Package Artifact Verification Baseline

`v0.33.1` continues the `0.33.x` stabilization stage on the road to `1.0.0`.

The previous patch recorded the first API stability audit baseline and added
the quick-start and API audit documents to the npm package file list. This
patch hardens the package verification script so that future package-facing
docs cannot drift out of the npm artifact check.

This version updates:

- `scripts/verify-package.mjs`
- `tests/package-boundary.test.mjs`
- `docs/api-stability-audit.md`

The package verification script now:

- uses `collectRequiredPackageFiles()` to derive required package files from
  `package.json.files`
- keeps exported JavaScript and declaration targets checked through the export
  map
- rejects development-only directories from the packed artifact
- rejects `docs/version/` release-history files from the npm package

## Product Boundary

This remains frontend 2D game engine package work.

`v0.33.1` does not add new runtime API, framework API, browser runtime behavior,
project generators, CLI scaffolds, visual editors, launchers, galleries,
template marketplaces, WeChat SDK wrappers, account systems, ads, monetization,
analytics services, publishing workflows, asset manager UIs, content authoring
workflows, HUD layout engines, audio mixers, galgame scripting languages, or
game-specific rules engines.

Game-specific rules still stay in downstream examples.
