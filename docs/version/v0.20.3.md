# leaferGame v0.20.3

## Version Positioning

`v0.20.3` closes the `0.20.x` stage:

**Collision Query Runtime Boundary Closeout**

`v0.20.1` added collision pair query APIs, and `v0.20.2` exposed those pairs through read-only tooling snapshots. This version makes `examples/dodge-blocks` consume that collision tooling path and documents the completed collision query runtime boundary.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.20.3` does not add physics simulation, collision response, trigger scripting, collider editing, map-generated colliders, editor controls, asset management, or content production workflows. It only proves that a downstream-style browser game can consume collision query observability from the engine package.

---

## Goals

After `v0.20.3`, the project should support:

- dodge-blocks passing `collisions: true` into `createToolingSnapshot(...)`
- browser tooling panel visibility for current / enter / stay / exit collision pair summaries
- example docs describing collision tooling as read-only runtime observability
- public docs describing `0.20.x` as a completed collision query runtime primitive stage
- package-boundary tests that keep collision tooling separate from editor/physics scope

---

## Scope

This version includes:

- dodge-blocks example tooling snapshot consumption
- example README updates
- README/public API/roadmap progress updates
- `v0.20.0` stage closeout update
- package-boundary tests for the example and docs

This version does not include:

- new collision runtime APIs
- new physics behavior
- collider authoring UI
- collision pair selection UI
- trigger callbacks
- map-to-collider generation

---

## Semantics

- Collision pair tooling remains opt-in through `createToolingSnapshot(scene, { collisions: true })`.
- The example only observes collision state; it does not mutate colliders, entities, systems, or scene config from tooling.
- Gameplay collision behavior remains driven by `CollisionSystem.hasCollision(...)`.
- `0.20.x` is considered complete after query APIs, tooling snapshots, and example consumption are all documented and tested.
