# leaferGame v0.31.0

## Version Positioning

`v0.31.0` starts the next stage:

**UI / Dialogue / Scene Flow Sprint**

`0.30.x` closed the drag/drop and selection hardening baseline. The next 1.0
gap is lightweight UI/dialogue flow for galgame-style and interactive narrative
prototypes: screen-space prompts, dialogue text, choices, and scene/flow
transitions that remain game-owned but are easier to assemble from package
helpers.

This version is a planning and boundary document. It does not add public package
API by itself.

---

## Why This Stage

The engine already has:

- screen-space HUD text through `createHudText(...)`
- `GameFlow` for ready/running/paused/ended-style state
- runtime scenes, systems, and lifecycle helpers
- pointer position, button, picking, selection, drag, and source-target action
  primitives
- browser runtime assembly and responsive resize behavior
- playable examples that prove arcade, collection, and pointer puzzle loops

The missing behavior is a small narrative/UI flow baseline. A downstream game
should be able to represent dialogue lines, present choices, route a selected
choice back to game-owned flow, and transition scenes or phases without copying
a large ad hoc scene file.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.31.0` does not add a visual novel scripting language, branching story
editor, dialogue timeline editor, save-file migration framework, character or
background asset authoring workflow, visual UI builder, scene graph editor,
launcher, gallery, marketplace, WeChat SDK wrapper, account system, ads,
monetization, analytics service, mobile app shell, or publishing workflow.

This stage may add small data contracts and helpers for dialogue text, choice
state, screen-space prompt rendering, scene-flow transitions, and a narrative
example. It should not become an editor, script engine, or content production
workflow.

## Candidate Engine Gaps

### 1. Dialogue And Choice Data Contract

Interactive narrative games need a deterministic package-facing shape for the
current line and available choices.

Potential package-level shape:

- dialogue line id / speaker / text data
- choice id / label / optional next target data
- copied read-only snapshots for tests and examples

Keep out of package:

- full scripting language
- branching story editor
- localization management system

### 2. Choice State And Selection Helpers

Pointer-first narrative screens need to track which choice is available,
selected, or resolved without hardcoding a story format.

Potential package-level shape:

- current choice list and selected choice id
- immutable select/clear/resolve helpers
- validation that selected choices belong to the current prompt

Keep out of package:

- story graph mutation UI
- save/load project workflow
- analytics or progression services

### 3. Screen-Space Prompt Rendering

The project already has HUD text. Narrative examples need enough screen-space
prompt rendering to prove dialogue and choices can be displayed without a UI
builder.

Potential package-level shape:

- small helper built from existing render text primitives
- deterministic text node creation/update behavior
- example-owned layout and styling

Keep out of package:

- visual UI builder
- theme designer
- component library product

### 4. Scene Flow And Narrative Example

At least one example should prove a tiny interactive narrative flow as a
downstream consumer.

Potential example shape:

- routeable `examples/dialogue-choice` shell
- a few dialogue lines with two or three choices
- simple scene/phase transitions and read-only gameplay snapshot

Keep out of package:

- story authoring tool
- content marketplace
- reusable galgame product template

---

## Done When

By the end of `0.31.x`:

- dialogue and choice data can be represented through small package-facing
  contracts
- choice state can be selected/resolved deterministically
- a narrative example consumes package APIs through package-style imports
- UI layout and story content remain example-owned
- no visual editor, scripting language, story authoring tool, launcher, gallery,
  marketplace, SDK wrapper, monetization, or publishing workflow is added
- `npm run check`, `npm test`, `npm run build:example`, `npm run verify:package`,
  and `npm audit --audit-level=moderate` pass
