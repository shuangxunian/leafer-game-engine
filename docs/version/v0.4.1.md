# leaferGame v0.4.1

## Version Positioning

`v0.4.1` is the first `0.4.x` delivery pass:

**Debug Overlay Pass**

This version turns the richer debug snapshot from `v0.3.5` into a browser-visible development overlay.

---

## Goals

After `v0.4.1`, the project should support:

- a small browser overlay for runtime stats
- pure formatting helpers for debug snapshots
- opt-in mounting and cleanup
- example integration

---

## Scope

This version includes:

- `formatDebugSnapshot()`
- `BrowserDebugOverlay`
- sample boot integration
- tests for formatting

This version does not include:

- clickable scene inspector
- collision box drawing
- timeline/replay tooling
- editor panel UI

---

## Semantics

- The overlay is opt-in and browser-only.
- It is owned by the example boot flow and detached on scene destroy.
- It reads `DebugSnapshot`; it does not inspect engine internals directly.
- Formatting remains pure and testable.

---

## Done When

- sample mounts a visible debug overlay
- destroy cleanup removes the overlay and interval
- snapshot formatting has tests
- `npm run check` and `npm test` pass
