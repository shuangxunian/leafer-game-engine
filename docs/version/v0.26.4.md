# leaferGame v0.26.4

## Version Positioning

`v0.26.4` continues the `0.26.x` framework extraction stage:

**Random Position In Bounds Helper Baseline**

Both playable examples need to choose a top-left position for a runtime actor inside a rectangular area:

- `examples/dodge-blocks` picks hazard spawn positions inside a hazard spawn region.
- `examples/collect-stars` picks star positions inside the playfield.

This version extracts only the repeated package-generic math:

`randomPositionInBounds(...)`

It does not extract a spawn system, enemy system, collectable system, random service, level authoring workflow, or gameplay loop abstraction.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.26.4` adds a small runtime math helper for selecting a random top-left position within rectangular bounds while accounting for actor width/height and optional padding. It does not add a visual editor, spawn editor, prefab authoring UI, random table editor, scene hierarchy editor, launcher, gallery, marketplace, save/load project workflow, or content publishing workflow.

Spawn cadence, actor size, velocity, score effects, collection rules, enemy behavior, and level semantics remain example-owned.

## Goals

After `v0.26.4`, the framework should provide:

- a pure helper for choosing a random `{ x, y }` position inside bounds
- support for optional bounds origin
- support for actor width/height
- support for symmetric padding
- deterministic tests through injected random functions
- stable behavior when the actor is larger than the available bounded area
- usage in both playable examples

## Semantics

- Bounds default to origin `{ x: 0, y: 0 }` when omitted.
- Padding defaults to `0`.
- Size defaults to `{ width: 0, height: 0 }`.
- The returned top-left `x` is selected between `bounds.x + padding` and `bounds.x + bounds.width - size.width - padding`.
- The returned top-left `y` is selected between `bounds.y + padding` and `bounds.y + bounds.height - size.height - padding`.
- If the available range is smaller than the actor, the helper anchors to the minimum bound on that axis.
- Callers may inject `random()` for deterministic tests; the returned value must be finite and within `[0, 1]`.
