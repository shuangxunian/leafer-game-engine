# leaferGame v0.2.1

## 版本定位

`v0.2.1` 是 `0.2.x` 阶段里的第一批落地。

这一版解决的是：

- system 调度顺序显式化
- world 查询能力补齐
- component 依赖校验起步

可以把它理解为：

**Framework Foundation Pass**

---

## 版本目标

`v0.2.1` 完成后，framework 复用能力至少要先站住这几个底层抓手：

- system 不再依赖隐式注册顺序
- world 不再要求系统直接扫裸 `entities`
- 常见 component 缺失依赖时能尽早暴露

---

## 范围边界

这一版要做：

- `System` priority / ordering
- `World` query API
- component dependency validation

这一版不做：

- render layer
- 可复用状态流抽象
- 更复杂的 camera / collision 能力升级

---

## 核心交付物

### 1. System priority / ordering

目标：

- 让 system 执行顺序变成显式规则

完成标准：

- `priority` 生效
- 同 priority 保持稳定注册顺序
- `update` / `fixedUpdate` / `lateUpdate` 都遵守同一套排序

---

### 2. World query API

目标：

- 让 framework system 有稳定查询入口

建议 API：

- `getEntities()`
- `getEntitiesWith()`
- `getEntitiesWithAll()`
- `getEntitiesWithAny()`

完成标准：

- query 只返回当前 live entity snapshot
- query 语义与 pending add/remove queue 一致

---

### 3. Component dependency validation

目标：

- 让装配错误尽早暴露

第一批覆盖：

- `ViewComponent -> TransformComponent`
- `ColliderComponent -> TransformComponent`
- `ColliderComponent -> SizeComponent` 或显式宽高

完成标准：

- 缺依赖时明确报错
- 错误信息包含 component 名、entity 名、依赖名

---

## 验收清单

发布 `v0.2.1` 前，至少确认：

- system 顺序是否已经显式化
- world 查询是否已走公共 API
- 关键 component 缺依赖时是否会 fail fast

---

## 与后续版本的关系

做完 `v0.2.1` 之后，再进入：

- [v0.2.2](v0.2.2.md)

也就是：

- `0.2.1` 先把 framework 的“调度、查询、依赖”打底
- `0.2.2` 再把 framework 的“结构层和状态流”补齐
