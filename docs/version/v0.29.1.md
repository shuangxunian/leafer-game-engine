# leaferGame v0.29.1

## Version Positioning

`v0.29.1` adds the first implementation slice for the responsive Web runtime
stage:

**Render Scene Resize Contract**

`v0.29.0` identified resize behavior as the next 1.0 gap. This version gives
render scenes an explicit viewport resize contract before adding any browser
resize bridge.

---

## What Changed

- Added `RenderSceneViewport` as the copied viewport data shape.
- Added `createRenderSceneViewport(width, height)` for Node-safe viewport
  validation and copying.
- Added `getRenderSceneViewport(renderScene)` for Node-safe viewport snapshots.
- Added `RenderScene.resize(width, height)` to the render scene contract.
- Implemented `resize(...)` in the Leafer adapter by updating the wrapper
  viewport state and forwarding the new size to native Leafer `resize(...)` when
  mounted.
- Covered valid and invalid resize behavior in tests.

---

## Product Boundary

This is runtime/render contract work for a frontend 2D game engine dependency
package.

`v0.29.1` does not add a browser resize observer, responsive page builder,
layout editor, visual UI designer, mobile app shell, WeChat SDK wrapper,
launcher, gallery, marketplace, account system, ads, monetization, or publishing
workflow.

The next slice should add an opt-in browser resize bridge that can read a mount
container and call this render scene resize contract.
