# leaferGame v0.30.3

## Positioning

`v0.30.3` adds the third implementation slice for the drag/drop and selection
hardening stage:

**Source-Target Action Baseline**

Selection and drag helpers now have a small generic action envelope that can
represent a source entity, target entity, and action type. Downstream games keep
owning the rules that decide whether the action is allowed or blocked.

## What Changed

- Added `SourceTargetActionRef`, `SourceTargetAction`,
  `SourceTargetActionSnapshot`, and `SourceTargetActionValidationResult`.
- Added `createSourceTargetAction(type, source, target)` for direct entity
  source-target action data.
- Added `createSourceTargetActionFromSelection(type, state)` to turn a ready
  source-target selection into generic action data.
- Added `getSourceTargetActionSnapshot(action)` for copied deterministic
  snapshots without exposing live entity objects.
- Added `allowSourceTargetAction(action)` and
  `blockSourceTargetAction(action, reason)` for explicit allowed/blocked
  validation results.
- Added `isSourceTargetActionAllowed(result)` for generic result checks.

## Boundary

`v0.30.3` does not add drop target resolution, water-sort validation, match-3
validation, merge rules, inventory rules, scoring, level completion, undo/hint
systems, solvers, gesture recognition, multi-touch gameplay systems, physics
simulation, editor drag handles, visual layout tools, launcher, gallery,
marketplace, SDK wrapper, monetization, or publishing workflow.

## Next

The next slice should harden the pointer puzzle example so it consumes the
generic selection/drag/action helpers while keeping pour-sort rules in
`examples/pour-sort`.
