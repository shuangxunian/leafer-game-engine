# leaferGame v1.0.0

## Release Baseline

`v1.0.0` promotes the stabilized package/API boundary to the first release
baseline.

This release keeps the public entrypoint shape established during the `0.33.x`
stabilization stage. It does not add new runtime API. It records that the
current package is ready as a lightweight Web 2D game engine baseline for
building browser mini-games, pointer-first puzzle interactions, and simple
interactive narrative flows.

The 1.0 baseline includes:

- image-backed sprite rendering and asset-to-render handoff
- responsive browser runtime resize behavior
- pointer position, picking, selection, drag, and source-target helpers
- quick-start scene assembly helpers for input, systems, HUD, and audio runtime
- screen-space dialogue prompt and choice-flow primitives
- package-boundary checks for public entrypoints, Node-safe subpaths,
  browser-facing entrypoints, package metadata, and npm artifact files

This version updates:

- `package.json`
- `package-lock.json`
- `README.md`
- `docs/api-stability-audit.md`
- `docs/public-api.md`
- `docs/roadmap.md`
- `tests/package-boundary.test.mjs`

## Product Boundary

This remains frontend 2D game engine package work.

`v1.0.0` does not add new runtime API, framework API, browser runtime behavior,
project generators, CLI scaffolds, visual editors, launchers, galleries,
template marketplaces, WeChat SDK wrappers, account systems, ads, monetization,
analytics services, publishing workflows, asset manager UIs, content authoring
workflows, HUD layout engines, audio mixers, galgame scripting languages, or
game-specific rules engines.

This is not a mature commercial engine. It is the first stable baseline for a
small package that helps frontend developers assemble lightweight Web games.

Game-specific rules still stay in downstream examples.
