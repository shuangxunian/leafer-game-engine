# leaferGame v0.6.2

## Version Positioning

`v0.6.2` is the second `0.6.x` delivery pass:

**Browser Tooling Panel Shell Pass**

This version adds a browser-facing panel class that can render aggregated tooling snapshots.

---

## Goals

After `v0.6.2`, the project should support:

- browser-visible tooling snapshot output
- a panel shell separate from the compact debug overlay
- rendering formatted tooling snapshot lines into the DOM
- clean mount/update/detach lifecycle
- sample integration with richer tooling output

---

## Scope

This version includes:

- `BrowserToolingPanel`
- panel styling and DOM lifecycle
- `ToolingSnapshot` rendering through `formatToolingSnapshot`
- dodge-blocks integration using `createToolingSnapshot`

This version does not include:

- clickable entity rows
- collapsible sections
- component editing
- schema visualization
- persistent panel settings

---

## Semantics

- `BrowserDebugOverlay` remains valid and unchanged.
- `BrowserToolingPanel` is a richer companion surface for `ToolingSnapshot`.
- The panel renders plain text lines for now.
- The panel owns its DOM node and removes it on `detach()`.
- The sample uses the richer panel so inspector data becomes visible.

---

## Done When

- A `ToolingSnapshot` can be rendered in a browser panel.
- The panel can mount, update, and detach.
- The dodge-blocks example uses `BrowserToolingPanel`.
- `npm run check`, `npm test`, and `npm run build:example` pass.
