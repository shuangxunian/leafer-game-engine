# leaferGame v0.17.1

## Version Positioning

`v0.17.1` is the first implementation pass in the `0.17.x` stage:

**Phase Cleanup On Runtime Errors**

This version hardens `Scene` and `World` update phases so pending entity mutations are flushed even when a system or component throws during `update`, `fixedUpdate`, or `lateUpdate`.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.17.1` does not add editor UI, runtime mutation controls, browser devtools, visual debugging controls, app routing, save/load systems, or content authoring workflows. It only strengthens core runtime invariants.

---

## Goals

After `v0.17.1`, the project should support:

- `Scene.update(...)`, `fixedUpdate(...)`, and `lateUpdate(...)` always closing their world phase
- `World.update(...)`, `fixedUpdate(...)`, and `lateUpdate(...)` always closing their nested phase
- thrown system/component errors still propagating to callers
- pending entity additions/removals flushing after failed phases
- regression tests proving phase cleanup after system and component errors

---

## Scope

This version includes:

- `try/finally` phase cleanup in `Scene`
- `try/finally` phase cleanup in `World`
- focused core tests for thrown runtime hooks
- README/public API progress updates

This version does not include:

- error swallowing or recovery policies
- global error callbacks
- runtime error overlay UI
- system retry behavior
- scene restart behavior

---

## Semantics

- Runtime hook errors still throw.
- The engine guarantees phase cleanup before the error exits the update call.
- Entity mutations requested before the error are flushed at the normal outermost phase boundary.
- No new editor or runtime control surface is introduced.

---

## Done When

- A system that creates an entity and throws during `Scene.update(...)` does not leave world phase state stuck.
- A component that creates an entity and throws during `World.update(...)` does not leave nested world phase state stuck.
- Existing lifecycle and mutation tests still pass.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
