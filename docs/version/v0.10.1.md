# leaferGame v0.10.1

## Version Positioning

`v0.10.1` is the first `0.10.x` delivery pass:

**Public API Inventory And Import Smoke Tests**

This version records the current package entrypoint boundary and adds smoke tests for package-name imports that should work in Node.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.10.1` does not add editor UI, visual scene editing, asset authoring, or new gameplay scope. It only clarifies and tests how downstream projects should import the engine package.

---

## Goals

After `v0.10.1`, the project should support:

- a documented public API inventory
- package-name smoke tests for Node-safe subpath imports
- a clear distinction between Node-safe imports and browser-facing imports
- package export map checks against `package.json`
- a foundation for later root/subpath import guidance

---

## Scope

This version includes:

- `docs/public-api.md`
- `tests/package-boundary.test.mjs`
- `v0.10.0` stage-plan link update
- README current-progress update
- roadmap recent-version link update

This version does not include:

- root entrypoint redesign
- new package subpaths
- browser runtime refactor
- npm publishing
- editor or asset browser work

---

## Semantics

- Node-safe package smoke tests should import the package by its real package name, not by relative `../lib/...` paths.
- Browser-facing entrypoints remain covered by browser/example builds for now.
- The test suite should make package boundary regressions easier to catch before publishing.

---

## Done When

- Public API inventory docs exist.
- `@shuangxunian/leafer-game-engine/core`, `/framework`, and `/tooling` package-name imports are smoke-tested.
- The package export map is checked in tests.
- README and roadmap point to `v0.10.1`.
- `npm run check`, `npm test`, and `npm run build:example` pass.
