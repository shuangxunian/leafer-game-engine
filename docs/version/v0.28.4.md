# leaferGame v0.28.4

## Version Positioning

`v0.28.4` adds the fourth implementation slice for the real sprite / image
rendering stage:

**Sprite Rendering Package Boundary**

`v0.28.1` made `source` visible through the Leafer adapter, `v0.28.2` made the
asset-to-render handoff explicit, and `v0.28.3` proved the path in a playable
example. This version packages the boundary so downstream consumers can see what
is supported before `0.28.x` closes.

---

## What Changed

- Added `docs/sprite-rendering.md` as a package-facing boundary document.
- Documented that `RenderSpriteAsset.source` maps to Leafer `Image.url`.
- Documented `AssetRegistry.getSpriteRenderAsset(id)` and
  `AssetRegistry.requireSpriteRenderAsset(id)` as the render-safe asset handoff.
- Documented that `SpriteAnimationSystem` and `attachActorSpriteView(...)`
  consume render-safe sprite assets through package-facing APIs.
- Added the sprite rendering boundary doc to the npm package file list.
- Updated package docs and roadmap links for `v0.28.4`.

No runtime behavior changes are added in this slice.

---

## Product Boundary

This is package documentation and publish-boundary work for a frontend 2D game
engine dependency package.

`v0.28.4` does not add public package API. It does not add an asset editor,
visual asset manager, atlas packer, sprite editor, image cropper, CDN pipeline,
bundled art library, marketplace, launcher, gallery, WeChat SDK wrapper, account
system, ads, monetization, or publishing workflow.

`/framework` remains DOM-free: it passes ids, dimensions, fills, frame metadata,
animation state, and `source` strings, not browser `Image` objects or decoded
bitmap resources.

The next `0.28.x` slice should decide whether the real sprite / image rendering
stage is ready to close.
