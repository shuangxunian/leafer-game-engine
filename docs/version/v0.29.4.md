# leaferGame v0.29.4

## Positioning

`v0.29.4` adds the fourth implementation slice for the responsive Web runtime
stage:

**Example Responsive Verification**

`v0.29.1` gave render scenes an explicit resize contract, `v0.29.2` added the
opt-in browser resize bridge, and `v0.29.3` made browser pointer local
coordinates read the latest mount bounds after resize. This version wires that
runtime behavior into the browser examples so the example route and standalone
`dodge-blocks` entry no longer assume one fixed initial canvas size.

## What Changed

- Enabled `resize: true` in the shared `examples/main.ts` browser runtime entry.
- Enabled `resize: true` in the standalone `examples/dodge-blocks/main.ts`
  browser runtime entry.
- Documented the minimal example boot shape with the opt-in resize bridge.
- Added package-boundary verification that the example entries opt into resize
  while keeping responsive layout choices example-owned.

The examples still use their own CSS and gameplay code for layout decisions.
The runtime bridge only observes the mount container and synchronizes render
scene viewport state, so a narrow 320x240-ish container or a taller
414x736-ish mobile viewport exercises the same package contract instead of a
new platform shell.

## Boundary

`v0.29.4` does not add a responsive page builder, polished launcher, gallery,
design system, mobile app shell, WeChat SDK wrapper, publishing workflow,
marketplace, account system, ads, monetization, visual layout editor, or
game-specific responsive rules in the package.

## Next

The next slice should close the `0.29.x` responsive runtime stage if the
evidence is enough: resize contract, browser bridge, pointer local coordinates,
and example verification are now all represented.
