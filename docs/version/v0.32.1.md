# leaferGame v0.32.1

## Scene Runtime Preset Baseline

`v0.32.1` continues the `0.32.x` Quick-Start Game Kit stage.

`v0.32.0` reduced browser input bridge boot glue. This patch reduces the next
small repeated setup step inside scenes: installing common scene-owned runtime
systems.

This version adds:

- `createSceneRuntimePreset(scene, options?)`
- `SceneRuntimePreset`
- `SceneRuntimePresetOptions`

The helper can install and return:

- `InputSystem`
- `CollisionSystem`
- `RuntimeServicesSystem` plus its `RuntimeServices`

When called without options, it installs a small default runtime preset:

- input
- runtime services

When options are provided, only explicitly enabled systems are installed. The
helper reuses existing systems instead of adding duplicates, so examples and
downstream games can call it near scene setup without worrying about double
installing input, collisions, or runtime services.

---

## Example Consumption

The routed examples now consume `createSceneRuntimePreset(...)` for repeated
scene-owned system setup:

- `examples/collect-stars` installs input, collisions, and runtime services
- `examples/dodge-blocks` installs input and collisions before its example-owned
  audio runtime setup
- `examples/pour-sort` installs input and passes it to the example-owned
  selection system
- `examples/dialogue-choice` installs input before its example-owned dialogue
  shell system

Gameplay rules, input action maps, HUD layout, audio cue choices, scene config,
dialogue prompt routing, pour-sort validation, scoring, timers, and snapshots
remain owned by the examples.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.32.1` does not add a project generator, CLI scaffold, visual editor,
launcher, gallery, template marketplace, WeChat SDK wrapper, account system,
ads, monetization, analytics service, mobile app shell, publishing workflow,
game-specific rules engine, water-sort solver, match-3 solver, galgame script
format, statechart authoring tool, UI builder, or content authoring workflow.

The helper only installs existing scene-owned runtime systems. It does not
create gameplay entities, define game rules, mutate content files, generate a
project, or introduce a new app surface.
