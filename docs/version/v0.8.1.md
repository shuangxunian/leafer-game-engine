# leaferGame v0.8.1

## Version Positioning

`v0.8.1` is the first `0.8.x` delivery pass:

**Asset Loader Abstraction And Load State Model**

This version adds a pure async asset loading abstraction and stable load-state data to `AssetRegistry`.

---

## Goals

After `v0.8.1`, the project should support:

- async sprite asset loader functions
- per-sprite load state
- deterministic load result data
- failed-load error reporting
- skipped-load behavior for already loaded sprites

---

## Scope

This version includes:

- `SpriteAssetLoader`
- `AssetLoadStatus`
- `AssetLoadState`
- `AssetRegistry.loadSprite(...)`
- `AssetRegistry.getSpriteLoadState(...)`
- tests for success, failure, missing asset, and already-loaded behavior

This version does not include:

- async manifest loading
- browser `Image` loading
- loading progress percentages
- asset dependency graphs
- audio or atlas loading

---

## Semantics

- Registering a sprite stores metadata but does not imply the sprite is loaded.
- Loading a sprite requires an explicit loader function.
- Successful loads mark the sprite as `loaded`.
- Failed loads mark the sprite as `failed`.
- Loading an already loaded sprite returns a skipped result.
- Missing sprite ids fail clearly without mutating registry state.

---

## Done When

- `AssetRegistry` exposes explicit sprite load state.
- `loadSprite(...)` can resolve loaded, failed, and skipped outcomes.
- Existing manifest registration behavior remains compatible.
- `npm run check`, `npm test`, and `npm run build:example` pass.
