# leaferGame v0.29.3

## Version Positioning

`v0.29.3` adds the third implementation slice for the responsive Web runtime
stage:

**Pointer Coordinates After Resize**

`v0.29.2` added an opt-in browser resize bridge. This version makes browser
pointer position mapping explicit for mount-local / viewport-local coordinates
so pointer-first examples can keep entity picking aligned after the mount bounds
change.

---

## What Changed

- Added `createBrowserPointerLocalPositionResolver(target)` to the Node-safe
  `framework` entrypoint.
- Added `getBrowserPointerLocalPosition(event, target)` for deterministic
  coordinate conversion tests.
- The resolver reads `target.getBoundingClientRect()` on every pointer event, so
  it uses the latest browser mount bounds after resize.
- Updated `examples/pour-sort` to consume the shared resolver instead of
  hand-rolling `clientX/clientY - rect.left/top` in example boot code.

---

## Product Boundary

This is pointer coordinate plumbing for a frontend 2D game engine dependency
package.

`v0.29.3` does not add drag/drop state, gesture recognition, multi-touch
gameplay systems, editor selection handles, responsive layout rules, a mobile
app shell, a WeChat SDK wrapper, launcher, gallery, marketplace, account system,
ads, monetization, or publishing workflow.

The next slice should verify at least one playable example in desktop and
mobile-ish viewport shapes without turning the example route into a polished
launcher or responsive page builder.
