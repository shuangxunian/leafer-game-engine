# leaferGame v0.19.3

## Version Positioning

`v0.19.3` is the third implementation pass in the `0.19.x` stage:

**Dodge Blocks Pointer Action Consumption**

The previous patches added pointer button action bindings and a browser pointer button bridge. This version wires those primitives into `examples/dodge-blocks` so the example validates a full browser path from pointer button events to `InputSystem`, `InputActionMap`, and gameplay actions.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.19.3` does not add pointer position tracking, drag/drop behavior, gestures, hit testing, runtime rebinding UI, settings screens, editor controls, or content authoring workflows. It only updates a downstream-style example to consume existing reusable pointer input APIs.

---

## Goals

After `v0.19.3`, the project should support:

- `examples/dodge-blocks` attaching `BrowserPointerButtonBridge`
- `confirm` accepting primary pointer button input through `InputActionMap`
- example docs describing pointer confirm without implying UI/editor scope
- package-boundary tests proving the example consumes pointer bridge APIs through package-style imports
- existing input action tooling continuing to show pointer button bindings as read-only state

---

## Scope

This version includes:

- dodge-blocks boot integration for browser pointer button bridge
- dodge-blocks action map update for primary pointer confirm
- example README updates
- package-boundary test coverage for example pointer input consumption

This version does not include:

- pointer move or pointer position
- aiming, drag, swipe, or gesture gameplay
- mouse hover state
- runtime input rebinding UI
- editor or authoring workflows

---

## Semantics

- `confirm` remains a semantic gameplay action.
- `Space` and `Enter` remain valid confirm bindings.
- Primary pointer button input also triggers `confirm`.
- The example still reads semantic actions in gameplay code instead of raw physical input ids.
- Pointer bridge cleanup follows scene destroy, the same way keyboard bridge cleanup does.

---

## Done When

- Dodge Blocks attaches and detaches `BrowserPointerButtonBridge`.
- Dodge Blocks `confirm` includes `definePointerButtonBinding("primary")`.
- Example docs mention primary pointer confirm.
- Package-boundary tests cover example pointer bridge and binding consumption.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
