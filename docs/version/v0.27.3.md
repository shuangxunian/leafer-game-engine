# leaferGame v0.27.3

## Version Positioning

`v0.27.3` is the third implementation pass in the `0.27.x` pointer-first puzzle interaction stage:

**Source-Target Selection State Baseline**

`v0.27.1` added pointer position state. `v0.27.2` added rectangle hit testing and entity picking. This version adds the smallest reusable state pattern for games that select one runtime entity and then select another entity as the target.

This is for pointer-first mini-game runtime interaction, not editor selection.

## Scope

This version includes:

- immutable source-target selection state snapshots
- entity selection refs containing the entity object, id, and name
- source selection that clears any previous target
- target selection that requires a selected source
- default protection against selecting the same entity as both source and target
- an opt-in same-entity target path for games that need it
- target clearing while preserving the selected source
- copied source-target pair reads for downstream logic and tests

This version does not include:

- water-sort, matching, grouping, or pouring rules
- move validation
- drag/drop state
- gesture recognition
- visual selection handles
- editor picking or scene graph mutation UI
- undo, hints, progression, or level authoring

## Semantics

- `createSourceTargetSelectionState()` returns `{ phase: "empty" }`.
- `selectSourceTargetSource(state, entity)` returns a new state with `phase: "source-selected"`.
- Selecting a new source clears any previous target.
- `selectSourceTargetTarget(state, entity)` returns a new state with `phase: "target-selected"`.
- Target selection throws if no source is selected.
- Target selection throws when source and target are the same entity unless `allowSameEntity` is true.
- `clearSourceTargetTarget(state)` preserves source selection and clears target selection.
- `clearSourceTargetSelection()` resets the selection to empty.
- `getSourceTargetSelectionPair(state)` returns copied refs when both source and target are present.
