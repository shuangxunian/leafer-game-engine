# leaferGame v0.28.1

## Version Positioning

`v0.28.1` adds the first implementation slice for the real sprite / image rendering stage:

**Image-Backed Leafer Sprite Adapter Baseline**

This version keeps the existing render adapter contract and makes the Leafer adapter consume image-backed sprite metadata through that contract.

---

## What Changed

- `LeaferRenderAdapter.createSprite(...)` now creates a Leafer `Image` node for sprite views.
- `RenderSprite.setAsset(...)` maps `RenderSpriteAsset.source` to the native Leafer `Image.url`.
- Fill-only sprite assets still use the existing fill/width/height/cornerRadius placeholder behavior.
- Applying a fill-only asset after an image-backed asset clears the previous image URL.
- Adapter behavior is covered by a focused Node test with browser canvas stubs.

This means downstream browser games can start passing image-like sprite sources through existing sprite asset metadata and have those sources reach the Leafer render node.

---

## Product Boundary

This is render adapter/runtime package work.

`v0.28.1` does not add or change public package API. It does not add an asset authoring API, visual asset manager, atlas packer, sprite editor, image cropper, bundled asset library, launcher, gallery, marketplace, WeChat SDK wrapper, account system, ads, monetization, or publishing workflow.

The remaining `0.28.x` work should continue proving the asset loading to render asset handoff and example image asset consumption without turning the engine package into a content production suite.
