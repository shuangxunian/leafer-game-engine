# leaferGame v0.18.5

## Version Positioning

`v0.18.5` closes the `0.18.x` stage:

**Level/Map Runtime Boundary Closeout**

The previous patches added tile map contracts, level layout metadata, scene config integration, and dodge-blocks example consumption. This version packages that work into a formal package-facing boundary document so downstream games can understand how to use level/map runtime primitives without mistaking them for editor features.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.18.5` does not add a visual map editor, drag-and-drop level authoring, tile painting, save/load workflows, map rendering, pathfinding, physics, collision generation, or tooling mutation controls. It only documents reusable runtime contracts, package boundaries, and example consumption.

Level/map data remains runtime configuration and helper data, not editor state.

---

## Goals

After `v0.18.5`, the project should have:

- a dedicated level/map runtime boundary document
- public API docs that point to the level/map boundary
- package artifact verification for the new document
- package-boundary tests proving the doc ships and preserves non-editor language
- README progress updated to show the `0.18.x` stage is complete
- `v0.18.0` stage documentation closed out

---

## Scope

This version includes:

- `docs/level-map.md`
- package `files` and package verifier updates
- package-boundary tests for the new document
- README/public API/roadmap/version progress updates

This version does not include:

- new runtime APIs
- new example behavior
- tooling visibility for level/map data
- editor or authoring workflows
