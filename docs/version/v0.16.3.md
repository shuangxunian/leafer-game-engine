# leaferGame v0.16.3

## Version Positioning

`v0.16.3` is the third implementation pass in the `0.16.x` stage:

**Render Layer And Lifecycle Contract**

This version makes render scene layer ownership and lifecycle responsibilities easier for downstream games to understand and verify. The engine already exposes `RenderScene.layers.background/world/ui/overlay`; this patch documents those responsibilities, adds a stable layer-order helper, and ensures the render/view contract ships with package artifacts.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.16.3` does not add editor UI, visual scene graph editing, component editing, layer authoring tools, asset management UI, render timeline controls, shader/material authoring, or content production workflows. It only clarifies package-level render scene contracts.

---

## Goals

After `v0.16.3`, the project should support:

- a documented render/view contract for `RenderNode`, `ViewComponent`, render layers, sprite-capable nodes, and render scene lifecycle
- a stable `RENDER_SCENE_LAYER_ORDER` constant from `@shuangxunian/leafer-game-engine/adapter/render-types`
- a reusable `getRenderSceneLayerNames(...)` helper from the same Node-safe render contract subpath
- tooling render snapshots using the shared layer order instead of ad-hoc `Object.keys(...)`
- package artifact verification that includes the render/view contract doc

---

## Scope

This version includes:

- render layer ownership docs
- render scene lifecycle docs
- adapter-facing layer order helpers exposed through a Node-safe render-types subpath
- tests for layer order and tooling snapshot output
- package file and verifier updates

This version does not include:

- render adapter redesign
- new render layers
- layer mutation APIs
- scene graph editing
- editor layer panels
- example gameplay changes

---

## Semantics

- `background` is for non-gameplay backdrop visuals.
- `world` is for gameplay/world-space nodes and camera-driven content.
- `ui` is for screen-space HUD and game UI.
- `overlay` is for top-level screen-space modal or status overlays.
- `RenderScene.destroy()` owns renderer-level cleanup.
- Game scenes that receive a `RenderScene` should call `renderScene.destroy()` during scene teardown unless another owner explicitly manages it.
- Tooling can read layer names for observability, but must not mutate layer content.

## Done When

- The render/view contract doc is included in package artifacts.
- README and public API docs reference the new render/view contract.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
