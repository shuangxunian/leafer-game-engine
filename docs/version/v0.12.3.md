# leaferGame v0.12.3

## Version Positioning

`v0.12.3` is the third `0.12.x` delivery pass:

**Runtime Services Scene Integration**

This version connects the `v0.12.1` event bus and `v0.12.2` scheduler through an optional scene system pattern.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.12.3` does not add a visual editor, global app container, dependency injection framework, browser UI, DOM-specific runtime panel, asset browser, scripting graph, or npm publishing. It only adds a small, opt-in way for scenes to own runtime coordination services.

---

## Goals

After `v0.12.3`, the project should support:

- a reusable `RuntimeServices` object containing `EventBus` and `RuntimeScheduler`
- a `RuntimeServicesSystem` that can advance the scheduler during scene updates
- helper APIs for adding and retrieving runtime services from a scene
- optional cleanup of listeners and scheduled tasks when the system is destroyed
- stable system priority so scheduled work can run before gameplay systems by default
- Node-side tests and package export coverage

---

## Scope

This version includes:

- `RuntimeServices`
- `createRuntimeServices(...)`
- `RuntimeServicesSystem`
- `addRuntimeServices(...)`
- `getRuntimeServices(...)`
- tests for scheduler advancement, service lookup, injected services, cleanup, and disabled scheduler updates
- README/public API updates
- `v0.12.0` stage progress update

This version does not include:

- modifying `Scene` to always own runtime services
- global singleton services
- editor service containers
- tooling snapshot integration
- browser UI
- scene config integration
- npm publishing

---

## Semantics

- Runtime services are opt-in per scene.
- `RuntimeServicesSystem` updates its scheduler in `update(dt)`.
- The default priority is `-250`, so scheduler callbacks run before input/gameplay/collision systems unless consumers override priority.
- Destroying the system clears event listeners and scheduled tasks by default.
- Consumers can inject existing services for tests or shared setup.
- Consumers can disable scheduler updates if they only want service ownership and cleanup.
