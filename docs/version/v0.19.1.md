# leaferGame v0.19.1

## Version Positioning

`v0.19.1` is the first implementation pass in the `0.19.x` stage:

**Pointer Button Action Binding Baseline**

This version extends `InputActionMap` with Node-safe pointer button bindings, so downstream games can map semantic actions such as `select` or `confirm` to pointer buttons without depending on DOM APIs.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.19.1` does not add browser pointer events, touch gesture systems, drag/drop authoring, runtime rebinding UI, keybinding settings screens, profile persistence, gamepad input, editor controls, or browser control panels. It only adds reusable input binding primitives and read-only formatting support.

---

## Goals

After `v0.19.1`, the project should support:

- `PointerButton` values for primary, secondary, and auxiliary buttons
- `definePointerButtonBinding(...)`
- `normalizePointerButton(...)`
- stable raw input ids for pointer buttons
- `InputActionMap` binding, unbinding, listing, querying, and deduplication for pointer buttons
- read-only tooling formatting for pointer bindings

---

## Scope

This version includes:

- pointer button binding types in `/framework`
- pointer button normalization helpers
- pointer button action map tests
- tooling formatter support
- README/public API/input docs progress updates

This version does not include:

- browser pointer bridge
- pointer position tracking
- touch gestures
- example gameplay consumption
- settings UI or editor workflows

---

## Semantics

- Pointer buttons normalize to `primary`, `secondary`, or `auxiliary`.
- Raw pointer input ids use a deterministic `pointer:<button>` shape.
- Pointer bindings can be mixed with keyboard bindings in the same action.
- Duplicate pointer bindings are ignored the same way duplicate keyboard bindings are ignored.
- Tooling displays pointer bindings in a stable read-only format.
