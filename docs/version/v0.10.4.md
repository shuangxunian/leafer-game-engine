# leaferGame v0.10.4

## Version Positioning

`v0.10.4` is the fourth `0.10.x` delivery pass:

**Example Import Boundary Cleanup**

This version makes the dodge-blocks example import the engine through package-style entrypoints instead of direct `../../src/...` paths.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.10.4` does not add gameplay scope, editor UI, asset browser UI, or new package exports. It only makes the example behave more like a downstream consumer while keeping local development mapped to source files.

---

## Goals

After `v0.10.4`, the project should support:

- examples importing engine APIs through `@shuangxunian/leafer-game-engine` package-style entrypoints
- local Vite development resolving those package-style imports to `src`
- TypeScript checks resolving the same package-style imports to source files
- example docs explaining the package-style import boundary
- no direct `../../src/...` engine imports inside `examples/dodge-blocks`

---

## Scope

This version includes:

- Vite resolve aliases for package entrypoints
- TypeScript path mappings for package entrypoints
- dodge-blocks import cleanup
- example README update
- public API docs note for local example aliases
- `v0.10.0` stage-plan link update
- roadmap recent-version link update

This version does not include:

- changing published package exports
- moving examples into a separate package
- npm publishing
- root entrypoint redesign
- gameplay changes
- editor or asset browser work

---

## Semantics

- Example code should resemble a real consumer project.
- Local aliases are a repository development convenience only.
- Published consumers still resolve imports through `package.json` exports.
- Example-local files can still use relative imports for other example-local modules.

---

## Done When

- Dodge-blocks imports engine APIs through package-style entrypoints.
- No `../../src/...` imports remain in `examples/dodge-blocks`.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
