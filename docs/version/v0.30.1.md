# leaferGame v0.30.1

## Positioning

`v0.30.1` adds the first implementation slice for the drag/drop and selection
hardening stage:

**Selection State Helper Hardening**

`v0.27.3` introduced the source-target selection baseline. This version keeps
that model compatible while adding clearer package-facing helpers for downstream
pointer-first games that need snapshots, explicit replacement operations, and
readiness checks before resolving a game-owned action.

## What Changed

- Added `EntitySelectionSnapshotRef`, a copied selection reference containing
  only `entityId` and `entityName`.
- Added `SourceTargetSelectionSnapshot`, which reports `phase`, copied
  source/target refs, and `isReady`.
- Added `getSourceTargetSelectionSnapshot(state)` for read-only UI, tests, and
  downstream action planning without exposing live entity objects.
- Added `isSourceTargetSelectionReady(state)` for generic source-target
  readiness checks.
- Added `replaceSourceTargetSelectionSource(state, entity)` and
  `replaceSourceTargetSelectionTarget(state, entity, options)` as explicit
  immutable replacement operations.

Existing `createSourceTargetSelectionState(...)`,
`selectSourceTargetSource(...)`, `selectSourceTargetTarget(...)`,
`clearSourceTargetTarget(...)`, `clearSourceTargetSelection(...)`, and
`getSourceTargetSelectionPair(...)` remain compatible.

## Boundary

`v0.30.1` does not add drag state, drop target resolution, source-target action
validation, water-sort rules, match-3 rules, inventory rules, solver logic,
undo/hint systems, scoring, level completion, gesture recognition, multi-touch
gameplay systems, editor selection handles, visual layout tools, launcher,
gallery, marketplace, SDK wrapper, monetization, or publishing workflow.

## Next

The next slice should add a small drag state baseline for active entity identity,
start/current pointer position, completion, and cancellation.
