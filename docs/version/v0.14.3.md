# leaferGame v0.14.3

## Version Positioning

`v0.14.3` is the third implementation pass in the `0.14.x` stage:

**Dodge Blocks Runtime Observability Integration Polish**

This version aligns the dodge-blocks example documentation and verification with the runtime observability hardening work from `v0.14.1` and `v0.14.2`.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.14.3` does not add editor UI, gameplay scope, system toggle controls, component editing, scene graph editing, asset management UI, runtime mutation buttons, or npm publishing. It only clarifies and verifies that the example consumes read-only runtime observability APIs.

---

## Goals

After `v0.14.3`, the project should support:

- dodge-blocks docs that mention Runtime Debug panel system lifecycle/order visibility
- dodge-blocks docs that mention time and viewport runtime debug context
- tests that verify the example passes `game` and `renderScene` into `createToolingSnapshot(...)`
- tests that verify the example docs describe read-only runtime observability
- no gameplay or editor-surface changes

---

## Scope

This version includes:

- example README polish
- package-boundary/example verification tests
- `0.14.x` stage progress links

This version does not include:

- example gameplay changes
- new engine APIs
- browser editor controls
- component or scene mutation
- asset authoring UI
- npm publishing

---

## Semantics

- Dodge Blocks remains a downstream-style consumer of package APIs.
- The example's tooling panel remains read-only runtime observability.
- The Runtime Debug panel can show time, viewport, entity counts, system totals, system lifecycle, and system order from existing snapshots.
- Verification should inspect source/docs boundaries without coupling to browser DOM rendering.

---

## Done When

- Dodge Blocks docs describe the runtime observability integration clearly.
- Tests verify the example passes runtime debug context into tooling snapshots.
- Tests verify the example docs keep tooling framed as read-only observability.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
