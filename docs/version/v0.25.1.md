# leaferGame v0.25.1

## Version Positioning

`v0.25.1` is the first implementation pass in the `0.25.x` second playable example stage:

**Second Example Shell And Routing Baseline**

This version adds the second example shell and makes the local example entry able to boot either `dodge-blocks` or the new `collect-stars` shell. The goal is build coverage and package-consumption discipline before adding the second gameplay loop.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.25.1` does not add an example launcher product, visual editor, template gallery, asset manager, drag-and-drop scene builder, level editor, save/load project workflow, or content publishing flow.

It only adds a minimal local development route so the repository can build and smoke-test more than one downstream-style example.

---

## Goals

After `v0.25.1`, the project should support:

- a second example directory under `examples/collect-stars`
- a minimal browser route for selecting `dodge-blocks` or `collect-stars`
- package-style imports in the new example
- a shell scene that consumes existing engine APIs without new framework abstractions
- docs that state the second example is not playable yet
- package-boundary tests that keep the new example visible

---

## Scope

This version includes:

- `examples/main.ts` as the local example entry
- a small `collect-stars` shell scene and boot function
- `collect-stars` README coverage
- index page wiring for `?example=dodge-blocks` and `?example=collect-stars`
- package-boundary coverage for route/build visibility and package-style imports

This version does not include:

- the collect-stars gameplay loop
- collectible spawning
- scoring
- hazards, timers, or win/lose states
- audio cues
- tooling panel wiring for the second example
- generic framework API additions
- a polished example launcher UI

---

## Semantics

- `dodge-blocks` remains the default example route.
- `collect-stars` is a shell baseline, not a completed playable game.
- The shell can create a scene, install a basic input system, create screen-space HUD text, and render a simple tile-backed playfield.
- The route exists only to make local dev and browser build coverage exercise multiple examples.
- Any reusable framework abstraction should wait until the second example exposes real repeated friction.

---

## Done When

- `npm run build:example` includes the shared example entry and both example boot paths.
- `examples/collect-stars` exists with README and package-style imports.
- Package-boundary tests cover the new route, docs, and import discipline.
- README and public API docs mention `v0.25.1`.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
