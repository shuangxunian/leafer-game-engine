# leaferGame v0.14.2

## Version Positioning

`v0.14.2` is the second implementation pass in the `0.14.x` stage:

**Runtime Panel Readability Polish**

This version makes the browser-facing runtime debug panel section easier to scan while keeping it fully read-only.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.14.2` does not add editor UI, system toggle controls, component editing, scene graph editing, asset management UI, runtime mutation buttons, or npm publishing. It only changes how existing runtime debug snapshot data is presented in tooling panel sections.

---

## Goals

After `v0.14.2`, the project should support:

- a more readable Runtime Debug panel section
- clearer entity and system summary rows
- explicit system order rows that include priority, lifecycle, and runtime flags
- continued deterministic Node-side tests for panel section output
- no mutation behavior from tooling panel helpers

---

## Scope

This version includes:

- runtime debug panel section line formatting polish
- system order presentation tailored for browser tooling panel sections
- tests for the new panel section output
- README, roadmap, and stage doc progress updates

This version does not include:

- changes to `formatDebugSnapshot(...)`
- new runtime or framework APIs
- system enable/disable controls
- browser editor controls
- component or scene mutation
- npm publishing

---

## Semantics

- `createRuntimeDebugPanelSection(...)` may format lines differently from `formatDebugSnapshot(...)` because it targets browser panel readability.
- The panel section consumes existing snapshot data only.
- Tooling panel helpers remain deterministic and testable in Node.
- Snapshot creation and panel section formatting remain read-only.

---

## Done When

- Runtime Debug panel sections expose grouped, readable runtime/system rows.
- Existing aggregate tooling sections use the improved runtime panel section.
- Tests cover runtime panel section output with and without systems.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
