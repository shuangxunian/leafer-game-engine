# leaferGame v0.32.4

## Scene Audio Runtime Bundle Baseline

`v0.32.4` continues the `0.32.x` Quick-Start Game Kit stage.

The earlier quick-start patches reduced repeated input, runtime preset, scene
quick-start, and HUD text setup. This patch reduces the next small repeated
scene setup step for games that need semantic audio intent: installing or
reusing the scene-owned audio runtime system.

This version adds:

- `createSceneAudioRuntimeBundle(scene, options?)`
- `SceneAudioRuntimeBundle`

The helper returns the scene-owned `AudioRuntimeSystem` plus its
`AudioRuntimeState`. If the scene already has an `AudioRuntimeSystem`, the
helper reuses it instead of adding a duplicate. If not, it installs one with the
provided `AudioRuntimeSystemOptions`.

Downstream games still own the audio manifest, cue ids, channel choices, when
to play/stop/pause/resume cues, and whether to attach browser playback. Browser
playback remains explicit through `AudioPlaybackSystem` and
`BrowserAudioPlaybackAdapter`.

---

## Example Consumption

`examples/dodge-blocks` now consumes `createSceneAudioRuntimeBundle(...)` for
its scene-owned audio runtime setup while keeping its placeholder audio
sources, semantic cue ids, cue trigger timing, browser playback adapter setup,
and tooling snapshot choices example-owned.

`examples/collect-stars`, `examples/pour-sort`, and `examples/dialogue-choice`
do not opt into audio runtime setup in this patch.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.32.4` does not add an audio mixer, audio editor, waveform editor, playback
buttons, volume sliders, asset browser, Web Audio graph builder, project
generator, CLI scaffold, visual editor, launcher, gallery, template
marketplace, WeChat SDK wrapper, account system, ads, monetization, analytics
service, mobile app shell, publishing workflow, game-specific rules engine,
water-sort solver, match-3 solver, galgame script format, statechart authoring
tool, or content authoring workflow.

The helper only groups existing scene-owned audio runtime setup. It does not
create audio content, choose cues, install browser playback, define game rules,
mutate content files, generate a project, or introduce a new app surface.
