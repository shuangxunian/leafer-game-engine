# leaferGame v0.17.4

## Version Positioning

`v0.17.4` closes the `0.17.x` stage:

**Runtime/Game Loop Hardening Closeout**

This version consolidates the runtime hardening work from `v0.17.1` through `v0.17.3`, updates package-facing docs, and leaves the project ready for the next engine stage.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.17.4` does not add editor UI, runtime mutation controls, visual scene graph tools, browser devtools, app-shell routing, save/load systems, npm publishing, or content production workflows. It only closes the runtime/game loop hardening stage and documents the current package-facing guarantees.

---

## Stage Summary

`0.17.x` now covers:

- scene/world phase cleanup when systems or components throw during runtime phases
- deterministic `Game.tick(...)` error propagation
- fixed-step accumulator cleanup when a fixed update throws
- browser runtime start/stop ownership semantics through a reusable runtime controller
- runtime ownership documentation for `Game`, `Scene`, `World`, `RenderScene`, browser runtime, tooling, and downstream error policy

---

## Scope

This version includes:

- `v0.17.0` stage completion update
- README progress update from in-progress to closed
- public API boundary update for completed runtime/game loop hardening
- roadmap link to this closeout version
- package boundary tests that keep `0.17.x` docs discoverable

This version does not include:

- new runtime behavior beyond existing `0.17.x` work
- public API split changes
- browser UI changes
- editor-facing workflows
- publishing automation

---

## Closed Guarantees

After `0.17.x`, downstream games can rely on:

- runtime hook errors still propagating to callers
- scene/world phase cleanup running before thrown errors exit the phase
- attempted fixed-step slices being removed from the accumulator when `fixedUpdate(...)` throws
- `runtime.stop()` stopping the frame loop without destroying the active scene or mounted render scene
- tooling remaining read-only runtime observability, not an editor/control surface
