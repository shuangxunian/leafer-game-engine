# leaferGame v0.8.5

## Version Positioning

`v0.8.5` closes the `0.8.x` delivery stage:

**Dodge Blocks Asset Loading Integration And Docs Cleanup**

This version connects the dodge-blocks example to the async asset loading path and updates public docs so the resource loading sprint has a clear end state.

---

## Product Boundary

This remains engine package work.

The example is only a consumer and integration sample. It should prove the engine package's asset loading and tooling APIs are usable from a game project, without turning this repository into an editor or asset authoring product.

---

## Goals

After `v0.8.5`, the project should support:

- dodge-blocks preloading its manifest through `loadManifestAsync(...)`
- browser image loader usage from an example consumer
- loaded asset states appearing in the browser tooling panel
- example docs that explain the asset pipeline role
- `0.8.x` stage docs that describe the completed resource loading baseline

---

## Scope

This version includes:

- example-level async asset preload before runtime start
- inline example sprite sources suitable for browser image loading
- error handling for failed example preload
- README and example README updates
- `0.8.x` stage documentation cleanup

This version does not include:

- editor asset import UI
- asset browser panels
- external asset hosting
- atlas packing
- audio loading
- cache eviction
- publishing a new npm package version

---

## Semantics

- The example should use the same public framework asset APIs a consuming project would use.
- Preloading should happen before gameplay starts.
- Failed preload should fail clearly instead of silently starting a broken scene.
- Tooling should show asset states through the existing read-only runtime panel.
- Examples remain consumers, not engine implementation.

---

## Done When

- Dodge-blocks uses async manifest loading.
- The example can display loaded sprite state in tooling snapshots.
- Public docs state that `0.8.x` has completed the resource loading baseline.
- `npm run check`, `npm test`, and `npm run build:example` pass.
