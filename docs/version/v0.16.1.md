# leaferGame v0.16.1

## Version Positioning

`v0.16.1` is the first implementation pass in the `0.16.x` stage:

**Explicit View Sync Contract**

This version makes the existing `ViewComponent` transform/size synchronization behavior available as a named method, while keeping the existing `lateUpdate` behavior compatible.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.16.1` does not add editor UI, scene graph editing, component editing, runtime render-node mutation controls, visual inspectors, asset management UI, or content authoring workflows. It only clarifies reusable engine behavior for ECS-to-render synchronization.

---

## Goals

After `v0.16.1`, the project should support:

- a clear `ViewComponent.syncFromTransform(...)` method
- transform-only sync for position, rotation, and scale
- optional size sync when a `SizeComponent` is present
- existing `lateUpdate` sync behavior preserved
- tests proving sync behavior and node destruction semantics
- README/public API progress updates

---

## Scope

This version includes:

- `ViewComponent.syncFromTransform(transform, size?)`
- tests for explicit sync behavior
- docs noting the start of `0.16.x`

This version does not include:

- render layer refactors
- adapter API redesign
- sprite capability helper extraction
- render-node pooling
- editor controls
- example gameplay changes

---

## Semantics

- `ViewComponent` owns the render node passed to it and destroys that node when the component is destroyed.
- `syncFromTransform(...)` copies transform position, rotation, and scale into the render node.
- If size is provided, `syncFromTransform(...)` also copies width and height.
- If size is omitted, existing node width/height are left untouched.
- `lateUpdate(...)` continues to read `TransformComponent` and optional `SizeComponent` from the entity and delegates to `syncFromTransform(...)`.
