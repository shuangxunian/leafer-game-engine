# leaferGame v0.30.2

## Positioning

`v0.30.2` adds the second implementation slice for the drag/drop and selection
hardening stage:

**Entity Drag State Baseline**

This version adds a small framework-owned drag state helper for pointer-first
games that need to carry one active entity from pointer down through move,
completion, or cancellation. It keeps the package focused on generic drag
bookkeeping while downstream games continue to own drop rules and gameplay
actions.

## What Changed

- Added `EntityDragPointerPosition`, `EntityDragRef`,
  `EntityDragSnapshotRef`, `EntityDragState`, `EntityDragSnapshot`, and
  `EntityDragTransition`.
- Added `createEntityDragState()` for an idle drag state.
- Added `startEntityDrag(state, entity, pointerPosition)` to begin dragging an
  entity from a copied pointer-local position.
- Added `moveEntityDrag(state, pointerPosition)` to update the copied current
  pointer position without mutating prior drag state.
- Added `getEntityDragDelta(state)` and `getEntityDragSnapshot(state)` for
  read-only UI, gameplay planning, and tests without exposing live entity
  objects in snapshots.
- Added `isEntityDragActive(state)` for generic active-drag checks.
- Added `completeEntityDrag(state, pointerPosition?)` and
  `cancelEntityDrag(state, pointerPosition?)`, which return a clean idle state
  plus a deterministic completed or cancelled snapshot.

## Boundary

`v0.30.2` does not add drop target resolution, source-target action validation,
gesture recognition, multi-touch gameplay systems, pointer capture policy,
physics simulation, snapping, inertia, water-sort rules, match-3 rules,
inventory rules, solver logic, undo/hint systems, scoring, level completion,
editor drag handles, visual layout tools, launcher, gallery, marketplace, SDK
wrapper, monetization, or publishing workflow.

## Next

The next slice should add a small source-target action baseline so selected or
dragged source/target pairs can be represented as generic game-owned action
data.
