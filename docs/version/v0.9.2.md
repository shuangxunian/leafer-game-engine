# leaferGame v0.9.2

## Version Positioning

`v0.9.2` is the second `0.9.x` delivery pass:

**Dodge Blocks Consumes GameFlow**

This version migrates the dodge-blocks example from a local sample-specific phase state machine to the framework-level `GameFlow` primitive.

---

## Product Boundary

This remains engine package integration work.

The example should prove that a consumer game can use framework `GameFlow` for common gameplay phases. It should not add menu widgets, visual editor flows, or new gameplay scope.

---

## Goals

After `v0.9.2`, the project should support:

- dodge-blocks using `GameFlow` for ready/running/paused/ended phases
- unchanged player-facing controls and HUD behavior
- example docs that identify `GameFlow` as the gameplay phase primitive
- a clear path toward later flow tooling visibility

---

## Scope

This version includes:

- replacing local `StateMachine<Phase>` usage in dodge-blocks
- preserving start, pause, resume, gameover, and restart behavior
- updating example docs
- linking `v0.9.2` from the `0.9.x` stage plan

This version does not include:

- new gameplay features
- UI widget abstractions
- flow snapshot/tooling integration
- scene loading orchestration
- save/load flow

---

## Semantics

- The start screen maps to `GameFlow`'s `ready` phase.
- Active gameplay maps to `running`.
- Pause maps to `paused`.
- Game over maps to `ended`.
- Restart uses `GameFlow.start()` from `ended`.
- The example remains a consumer of framework APIs, not engine implementation.

---

## Done When

- Dodge-blocks no longer uses a local phase state machine for gameplay flow.
- Existing controls and HUD behavior remain compatible.
- Documentation describes the `GameFlow` integration.
- `npm run check`, `npm test`, and `npm run build:example` pass.
