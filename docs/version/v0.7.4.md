# leaferGame v0.7.4

## Version Positioning

`v0.7.4` is the fourth `0.7.x` delivery pass:

**Schema-Assisted Component Detail Formatting**

This version connects selected entity detail output with component schema metadata so runtime inspector details can explain component fields more clearly.

---

## Goals

After `v0.7.4`, the project should support:

- schema-aware selected entity detail lines
- component schema labels in selected component summaries
- field-level detail lines for known schema fields
- clear fallback behavior for components without schemas
- no runtime mutation or editor UI behavior

---

## Scope

This version includes:

- selected detail formatting that can receive `ComponentSchemaSnapshot`
- component-name to schema matching
- schema field rows showing type, required/default metadata, and current value
- tests for matched schema, unmatched component, and missing field values
- dodge-blocks docs update for schema-assisted selected details

This version does not include:

- component value editing
- form controls
- validation execution against live components
- custom gameplay component schemas
- editor UI inside this package

---

## Semantics

- Schema metadata is optional.
- If schema metadata is unavailable, selected detail output remains compatible with `v0.7.3`.
- Schemas are matched by component constructor name.
- Field rows show current primitive component data when available.
- Missing optional values are shown as `value=<unset>`.
- Missing required values are shown as `value=<missing>`.

---

## Done When

- Selected entity details can use schema metadata when available.
- Existing non-schema selected detail behavior remains stable.
- Tests cover schema match and fallback behavior.
- `npm run check`, `npm test`, and `npm run build:example` pass.
