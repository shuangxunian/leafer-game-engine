# leaferGame v0.17.3

## Version Positioning

`v0.17.3` is the third implementation pass in the `0.17.x` stage:

**Runtime Ownership Package Guidance**

This version documents and tests which runtime objects own scenes, render resources, frame loops, and read-only tooling state.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.17.3` does not add editor UI, runtime mutation controls, visual scene graph tools, asset managers, app-shell routing, save/load systems, npm publishing, or content production workflows. It only clarifies package-facing runtime ownership responsibilities.

---

After `v0.17.3`, the project should support:

- documented `runtime.start(...)` and `runtime.stop()` semantics

- tests proving browser runtime stop/start behavior does not accidentally destroy scene or render resources

## Semantics

- `Game` owns the active scene and destroys the previous scene when replacing it.
- `Scene` owns systems and world state.
- A scene may own a `RenderScene` only when downstream code passes that resource into the scene deliberately.
- `createBrowserRuntime(...)` owns the animation frame loop it creates.
- `runtime.start(scene)` starts the scene through `game.setScene(...)`, resets frame timing, and starts the loop.
- `runtime.stop()` stops the loop only; it does not destroy the active scene or mounted render scene.
- `tooling` remains read-only observability and does not own or mutate runtime resources.
