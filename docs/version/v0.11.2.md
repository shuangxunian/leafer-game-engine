# leaferGame v0.11.2

## Version Positioning

`v0.11.2` is the second `0.11.x` delivery pass:

**Animation Playback Timing Helpers**

This version adds deterministic sprite animation playback primitives on top of the `v0.11.1` frame and clip contracts.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.11.2` does not add a timeline editor, visual editor UI, asset browser UI, render-node animation system, ECS component, or example gameplay integration. It only adds pure framework helpers that downstream games and later engine systems can reuse.

---

## Goals

After `v0.11.2`, the project should support:

- creating an animation playback state from a `SpriteAnimationClip`
- advancing playback by deterministic delta seconds
- resolving the current frame id and frame index
- pausing, resuming, and stopping playback without mutating the previous state
- looping clips by default
- completing non-looping clips at the final frame
- using frame-level `durationSeconds` when frame metadata is supplied
- clear errors for empty clips, invalid delta time, missing timing data, or missing frame metadata

---

## Scope

This version includes:

---

## Semantics

- Playback helpers are pure functions and return new state objects.
- Clip `loop` defaults to `true`.
- Non-looping clips complete on the final frame and keep reporting that frame.
- Paused, stopped, and completed states do not advance when `advance...` is called.
- `stop...` resets elapsed time and frame index to zero.
- Frame-specific `durationSeconds` overrides clip-level `frameDurationSeconds` when frame metadata is provided.
- If neither a frame nor its clip provides positive timing data, playback fails clearly.
