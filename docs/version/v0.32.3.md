# leaferGame v0.32.3

## HUD Text Bundle Baseline

`v0.32.3` continues the `0.32.x` Quick-Start Game Kit stage.

The earlier quick-start patches reduced scene runtime and browser input bridge
setup. This patch reduces another small repeated setup step in downstream-style
games: creating several screen-space HUD text nodes one by one.

This version adds:

- `createHudTextBundle(renderAdapter, renderScene, items)`
- `HudTextBundle`
- `HudTextBundleItem`

The helper creates keyed HUD text nodes through the existing
`createHudText(...)` primitive. Downstream games still own every item id, text
string, coordinate, font size, visibility flag, and target layer. The bundle
only returns keyed nodes, a `get(id)` helper, and a `destroy()` helper for the
nodes it created. Duplicate ids fail before any nodes are created, so keyed HUD
references do not get silently overwritten.

---

## Example Consumption

Two routed examples now consume `createHudTextBundle(...)` where they already
created multiple HUD text nodes:

- `examples/collect-stars` creates its title, score, and status nodes through a
  keyed bundle
- `examples/dodge-blocks` creates its score/status and overlay prompt nodes
  through a keyed bundle

Gameplay rules, HUD layout choices, text content, score/timer behavior, input
action maps, audio cue choices, scene config, puzzle validation, prompt routing,
tooling snapshots, and gameplay snapshots remain owned by the examples.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.32.3` does not add a HUD layout engine, theme system, widget library, UI
builder, visual editor, project generator, CLI scaffold, launcher, gallery,
template marketplace, WeChat SDK wrapper, account system, ads, monetization,
analytics service, mobile app shell, publishing workflow, game-specific rules
engine, water-sort solver, match-3 solver, galgame script format, statechart
authoring tool, or content authoring workflow.

The helper only groups existing HUD text creation. It does not create gameplay
entities, define game rules, mutate content files, generate a project, or
introduce a new app surface.
