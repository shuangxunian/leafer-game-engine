# leaferGame v0.15.1

## Version Positioning

`v0.15.1` is the first implementation pass in the `0.15.x` stage:

**Scene Config Validation Baseline**

This version adds a Node-safe validation helper for scene configs so downstream games can inspect config problems before bootstrap mutates a scene.

---

## Product Boundary

This remains frontend 2D game engine package work.

`v0.15.1` does not add editor UI, scene graph editing, component editing, asset management UI, content publishing, save-file migration, remote content pipelines, or npm publishing. It only adds reusable validation primitives for data-driven scene consumption.

---

## Goals

After `v0.15.1`, the project should support:

- a pure `validateSceneConfig(...)` helper
- deterministic validation result shape
- validation errors with stable codes, paths, and messages
- checks for malformed `assets`, `entities`, `components`, and `systems` sections
- checks for unknown entity component types when a registry is available
- checks for missing/unknown systems when system declarations are present
- tests proving validation does not mutate a scene, asset registry, or system registry

---

## Scope

This version includes:

- `SceneConfigValidationResult`
- `SceneConfigValidationError`
- `validateSceneConfig(config, options?)`
- registry lookup helpers needed by validation
- Node tests for valid, invalid, unknown component, unknown system, missing registry, and no-mutation behavior
- README/public API progress updates

This version does not include:

- changing `bootstrapSceneFromConfig(...)` semantics
- async validation
- asset file existence checks
- render-node config
- example gameplay changes
- editor controls

---

## Semantics

- Validation is pure with respect to scenes and registries.
- Validation does not register assets, create entities, add systems, or call system factories.
- Validation may call entity template component factories only if needed to reuse existing data validation behavior.
- Validation should return all structural errors it can find without throwing for expected invalid config data.
- Bootstrap remains the existing mutation boundary.

---

## Done When

- Scene config validation API is exported from `/framework`.
- Valid configs return `{ ok: true, errors: [] }`.
- Invalid configs return deterministic error objects without mutating runtime state.
- Tests cover validation behavior and existing bootstrap behavior still passes.
- `npm run check`, `npm test`, `npm run build:example`, and `npm run verify:package` pass.
