# leaferGame v0.31.3

## Positioning

`v0.31.3` adds the third implementation slice for the UI / dialogue / scene
flow stage:

**Screen-Space Prompt View Baseline**

This version adds a small package-facing prompt view helper built on top of the
existing HUD text helper. It creates screen-space text nodes for the current
dialogue line and available choices, and can update those nodes when the
current prompt changes.

## Public API

The `/framework` entrypoint now exports:

- `DialoguePromptViewOptions`
- `DialoguePromptView`
- `createDialoguePromptView(renderAdapter, renderScene, options)`

The helper creates one line text node and one choice text node per prompt
choice through the injected `RenderAdapter`, attaches those nodes to the
screen-space `ui` layer by default or `overlay` when requested, returns copied
prompt snapshots, and supports prompt replacement plus visibility toggling.

## Boundary

This is a screen-space prompt view baseline only. It does not add a visual UI
builder, layout engine, theme system, component library, visual novel scripting
language, branching story editor, dialogue timeline editor, story graph
traversal, localization management, save/load framework, launcher, gallery,
marketplace, SDK wrapper, monetization, analytics service, mobile app shell, or
publishing workflow.

Story content, exact layout, choice effects, scene switching, transitions,
backgrounds, character portraits, and visual styling remain example-owned or
downstream-game-owned until repeated usage proves a smaller package-level
helper belongs in the engine.

## Next

The next slice should add a narrative example shell that consumes the dialogue
data, choice state, and prompt view helpers through package-style imports.
